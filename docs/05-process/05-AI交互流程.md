## 一、会话创建流程

### 1.1 会话类型

| 类型 | 说明 | 创建方式 |
|:---|:---|:---|
| 咨询会话 | 无绑定植物，通用咨询 | 用户主动创建 |
| 植物会话 | 绑定特定植物 | 从咨询会话升级或直接创建 |

### 1.2 创建咨询会话

```mermaid
sequenceDiagram
    participant User as 用户
    participant MiniApp as 小程序
    participant Backend as 后端服务
    participant DB as 数据库

    User->>MiniApp: 点击"开始咨询"
    MiniApp->>Backend: POST /api/sessions {type: 'consultation'}
    Backend->>DB: INSERT sessions 表
    DB-->>Backend: 返回 session_id
    Backend-->>MiniApp: {session info}
    MiniApp-->>User: 进入会话页面
```

### 1.3 创建植物会话

```mermaid
sequenceDiagram
    participant User as 用户
    participant MiniApp as 小程序
    participant Backend as 后端服务
    participant DB as 数据库

    User->>MiniApp: 进入植物详情
    User->>MiniApp: 点击"AI诊断"
    MiniApp->>Backend: POST /api/sessions {type: 'plant', plantId: 'xxx'}
    Backend->>DB: INSERT sessions 表
    Backend->>DB: 查询植物信息、环境数据
    Backend-->>MiniApp: {session info, context}
    MiniApp-->>User: 进入会话页面
```

## 二、消息发送流程

### 2.1 流程图

```mermaid
sequenceDiagram
    participant User as 用户
    participant MiniApp as 小程序
    participant Backend as 后端服务
    participant AI as AI服务
    participant DB as 数据库

    User->>MiniApp: 输入消息/上传图片
    MiniApp->>Backend: POST /api/sessions/:sessionId/messages
    
    Backend->>DB: INSERT messages (user message)
    
    Backend->>AI: 调用AI分析
    
    par AI处理
        AI->>AI: 组装Prompt
        AI->>AI: 查询上下文（植物信息、环境数据、历史消息）
        AI->>AI: 调用大模型API
    end
    
    AI-->>Backend: AI响应
    
    alt 产生诊断结果
        Backend->>DB: INSERT diagnosis_cards
    end
    
    Backend->>DB: INSERT messages (ai message)
    Backend-->>MiniApp: {ai message, diagnosis?}
    MiniApp-->>User: 显示AI回复
```

### 2.2 请求示例

**文本消息**:
```json
POST /api/sessions/:sessionId/messages
{
  "contentType": "text",
  "content": "我的植物叶子发黄了怎么办？"
}
```

**图片消息**:
```json
POST /api/sessions/:sessionId/messages
{
  "contentType": "image",
  "imageUrls": ["https://cos.example.com/img1.jpg"]
}
```

### 2.3 AI响应示例

```json
{
  "code": 200,
  "data": {
    "messageId": "MSG_002",
    "role": "assistant",
    "contentType": "text",
    "content": "根据您描述的情况，叶子发黄可能是由以下原因造成的...",
    "diagnosisCard": {
      "diagnosisCardId": "DC_001",
      "analysisType": "text",
      "healthScore": 65,
      "issues": [...],
      "suggestions": [...]
    }
  }
}
```

## 三、会话升级流程

### 3.1 流程概述

咨询会话可以升级为植物会话，保留历史消息。

### 3.2 流程图

```mermaid
sequenceDiagram
    participant User as 用户
    participant MiniApp as 小程序
    participant Backend as 后端服务
    participant DB as 数据库

    User->>MiniApp: 在咨询会话中点击"绑定植物"
    MiniApp->>MiniApp: 选择植物
    User->>MiniApp: 确认升级
    MiniApp->>Backend: POST /api/sessions/:sessionId/upgrade
    Note over MiniApp,Backend: {plantId: 'xxx'}
    
    Backend->>DB: UPDATE sessions SET plant_id=?, type='plant'
    Backend->>DB: 查询植物信息、环境数据
    Backend-->>MiniApp: {session info, newContext}
    MiniApp-->>User: 升级成功，显示植物上下文
```

### 3.3 升级后的变化

| 变化 | 说明 |
|:---|:---|
| 会话类型 | consultation → plant |
| 植物绑定 | NULL → plant_id |
| AI上下文 | 增加：植物信息、环境数据、养护记录、诊断历史 |

## 四、诊断分析流程

### 4.1 分析类型

| 类型 | 触发条件 | 数据来源 |
|:---|:---|:---|
| 文本分析 | 用户发送文本描述 | 用户描述 + 植物信息 |
| 图片分析 | 用户上传图片 | 图片识别 + 植物信息 |
| 综合分析 | 文本 + 图片 | 多模态分析 |

### 4.2 诊断流程

```mermaid
flowchart TB
    Start[用户发送消息] --> CheckType{消息类型}
    
    CheckType -->|文本| TextAnalysis[文本分析]
    CheckType -->|图片| ImageAnalysis[图片识别]
    CheckType -->|文本+图片| MultiAnalysis[综合分析]
    
    TextAnalysis --> AssemblePrompt[组装Prompt]
    ImageAnalysis --> ImageRecognition[调用视觉模型]
    MultiAnalysis --> ImageRecognition
    
    ImageRecognition --> AssemblePrompt
    
    AssemblePrompt --> QueryContext[查询上下文数据]
    
    subgraph Context[上下文数据]
        PlantInfo[植物信息]
        EnvData[环境数据]
        CareRecords[养护记录]
        DiagnosisHistory[诊断历史]
    end
    
    QueryContext --> CallLLM[调用大模型]
    CallLLM --> ParseResult[解析结果]
    
    ParseResult --> CreateDiagnosisCard[创建诊断卡]
    CreateDiagnosisCard --> SaveMessage[保存AI消息]
    SaveMessage --> ReturnResult[返回结果]
```

## 五、快速分析流程

### 5.1 流程概述

用户可从植物详情页直接发起快速分析，无需创建会话。

### 5.2 流程图

```mermaid
sequenceDiagram
    participant User as 用户
    participant MiniApp as 小程序
    participant Backend as 后端服务
    participant AI as AI服务
    participant DB as 数据库

    User->>MiniApp: 植物详情页点击"快速分析"
    MiniApp->>MiniApp: 拍照/选择图片
    User->>MiniApp: 确认分析
    MiniApp->>Backend: POST /api/ai/analyze
    
    Backend->>DB: 查询植物信息
    Backend->>DB: 查询环境数据
    Backend->>AI: 调用AI分析
    AI-->>Backend: 分析结果
    
    Backend->>DB: INSERT diagnosis_cards
    Backend->>DB: INSERT sessions (自动创建)
    Backend->>DB: INSERT messages
    
    Backend-->>MiniApp: {diagnosisCard, sessionId}
    MiniApp-->>User: 显示诊断结果
```

### 5.3 与会话分析的区别

| 对比项 | 快速分析 | 会话分析 |
|:---|:---|:---|
| 入口 | 植物详情页 | 会话页面 |
| 会话创建 | 自动创建 | 用户主动创建 |
| 历史消息 | 无 | 有上下文 |
| 后续对话 | 可继续 | 可继续 |

## 六、消息列表查询流程

### 6.1 分页查询

```mermaid
sequenceDiagram
    participant MiniApp as 小程序
    participant Backend as 后端服务
    participant DB as 数据库

    MiniApp->>Backend: GET /api/sessions/:sessionId/messages?page=1&size=20
    Backend->>DB: SELECT messages ORDER BY created_at DESC LIMIT 20
    DB-->>Backend: 消息列表
    Backend-->>MiniApp: {messages, hasMore}
```

### 6.2 消息类型

| contentType | 说明 | 显示方式 |
|:---|:---|:---|
| text | 纯文本 | 文本气泡 |
| image | 图片 | 图片网格 |
| diagnosis | 诊断卡 | 卡片组件 |
| system | 系统消息 | 居中提示 |
