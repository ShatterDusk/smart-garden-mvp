## 命名规范

### 文件命名
```
{序号}-{简短功能描述}.md
```

- **序号**: 两位数，从 01 开始递增
- **功能描述**: 简洁明了，10-15 个字符
- **示例**: `01-初始化环境指标定义数据.md`

### 标题规范
每个 TODO 文件必须包含以下信息：

```markdown
# {功能名称}

## 元信息
- **状态**: 待实现 / 进行中 / 已完成 / 已放弃
- **优先级**: P0 / P1 / P2 / P3
- **提出时间**: YYYY-MM-DD
- **预计工时**: X 小时
- **涉及模块**: 前端 / 后端 / 数据库 / 全栈
```

---

## 工作流

### 1. 创建 TODO
当对话中确认将实现某个功能时：
1. 在 TODO 文件夹创建新文件
2. 按序号递增命名
3. 填写完整的实现方案
4. 更新本 README 的目录索引

### 2. 实现 TODO
当开始实现某个功能时：
1. 修改该 TODO 文件的状态为「进行中」
2. 完成后修改状态为「已完成」
3. 可选择将文件移至 archive/ 或保留

### 3. 放弃 TODO
当决定不实现某个功能时：
1. 修改该 TODO 文件的状态为「已放弃」
2. 记录放弃原因
3. 将文件移至 archive/ 文件夹
