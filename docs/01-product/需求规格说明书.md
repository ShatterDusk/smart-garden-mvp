## 一、文档概述

### 1.1 文档目的

本文档详细描述智能园艺助手微信小程序的功能需求、用户故事和非功能需求，为开发团队提供明确的需求基线。

**目标读者**:

- 产品经理：了解产品功能范围和业务逻辑
- 开发团队：明确技术实现需求和接口规范
- 测试团队：制定测试计划和验收用例
- 项目管理者：把控项目进度和范围边界

### 1.2 产品定位

智能园艺助手是一款面向植物养护爱好者的AI驱动型微信小程序，通过图像识别和自然语言处理技术，为用户提供植物健康诊断、养护建议和环境监测服务。

### 1.3 目标用户

| 用户类型  | 特征          | 核心需求           |
| :---- | :---------- | :------------- |
| 新手养护者 | 刚开始养植物，缺乏经验 | 简单易懂的健康诊断和养护指导 |
| 资深爱好者 | 有多株植物，追求专业  | 精细化管理、历史数据分析   |
| 忙碌上班族 | 时间有限，需要提醒   | 自动化监测、养护提醒     |

### 1.4 术语定义

| 术语   | 定义                               |
| :--- | :------------------------------- |
| 咨询会话 | 不绑定植物档案的临时AI对话                   |
| 植物会话 | 绑定特定植物档案的专属对话                    |
| 普通分析 | 仅使用当前消息和图片的AI分析                  |
| 深度分析 | 携带植物历史数据的AI分析                    |
| 诊断卡  | 结构化存储的AI诊断结果                     |
| 升级   | 将咨询会话转换为植物会话（修改类型并绑定植物档案），保留历史消息 |

### 2.3 会话咨询模块

#### 2.3.1 咨询会话

**需求编号**: CS-001\
**需求描述**: 用户可进行临时AI咨询\
**优先级**: P0

**功能要求**:

- 无需创建植物档案即可开始对话
- 支持文字消息
- 支持图片消息
- AI自动分析图片并生成诊断卡
- 支持保存诊断结果到植物档案（升级为植物会话）

**会话升级说明**:

当用户在咨询会话中获得诊断结果后，可选择将诊断保存到植物档案，此时触发**会话升级**流程：

**升级触发条件**:

1. 快速分析后点击"保存到档案"
2. 咨询会话中产生诊断卡后点击"保存到植物档案"

**升级流程**:

1. 用户点击"保存到档案"按钮
2. 跳转至创建植物档案页面（自动填充诊断信息）
3. 用户确认/修改植物信息，点击"创建并关联"
4. 后端调用升级API（POST /api/sessions/:id/upgrade）
5. 原咨询会话升级为植物会话（修改type和plant\_id）
6. 保留所有消息历史
7. 跳转至升级后的植物会话页面

**升级后变化**:

- 会话类型：`consultation` → `plant`
- 上下文开关：从不可用变为可用
- AI分析类型：`normal` → `deep`
- 历史消息：标记"升级前"标签

**验收标准**:

- [ ] 新用户首次使用可立即开始咨询
- [ ] 发送图片后10秒内返回分析结果
- [ ] 诊断卡正确显示健康评分和建议

### 2.4 AI诊断模块

#### 2.4.1 图像识别诊断

**需求编号**: AI-001\
**需求描述**: AI识别植物图片并诊断\
**优先级**: P0

**功能要求**:

- 识别植物品种
- 评估健康状态
- 检测潜在问题（浇水、光照、病虫害等）
- 生成结构化诊断卡

**诊断卡内容**:

| 字段                  | 类型       | 说明                                          |
| :------------------ | :------- | :------------------------------------------ |
| `diagnosis_card_id` | string   | 诊断卡唯一标识，UUID                                |
| `message_id`        | string   | 关联产生此诊断的消息ID                                |
| `plant_id`          | string   | 关联植物ID（咨询会话为空）                              |
| `analysis_type`     | enum     | 分析类型：`normal`/`deep`                        |
| `species`           | string   | 识别到的植物品种                                    |
| `health_score`      | int      | 健康评分 0-100                                  |
| `status`            | enum     | 健康状态：`healthy`/`warning`/`critical`         |
| `issues`            | array    | 问题列表，每项包含：type, name, severity, description |
| `suggestions`       | array    | 建议列表，每项包含：type, action, details             |
| `confidence`        | decimal  | 置信度 0.00-1.00                               |
| `context_used`      | object   | 实际使用的上下文数据                                  |
| `created_at`        | datetime | 诊断时间                                        |

**issues/suggestions 数组项结构**:

```json
{
  "type": "watering",      // 问题/建议类型
  "name": "浇水过多",       // 问题名称（issues）
  "action": "减少浇水",     // 建议行动（suggestions）
  "severity": "medium",     // 严重程度：low/medium/high
  "description": "...",     // 详细描述（issues）
  "details": "..."          // 详细说明（suggestions）
```

#### 2.5.2 环境数据监测

**需求编号**: DV-002\
**需求描述**: 实时显示环境数据\
**优先级**: P1

**功能要求**:

- 显示实时温度、湿度、光照、土壤湿度
- 显示数据状态（适宜/偏高/偏低）
- 支持查看历史趋势图
- 支持设备+天气数据混用

**监测指标**:

| 指标   | 单位     | 数据来源     |
| :--- | :----- | :------- |
| 温度   | °C     | 设备/天气API |
| 湿度   | %      | 设备/天气API |
| 光照   | lux/状态 | 设备       |
| 土壤湿度 | %      | 设备       |
| 土壤温度 | °C     | 设备       |

**数据源切换机制**:

环境数据支持多种数据源混合使用，优先级和切换规则如下：

| 数据类型 | 优先级        | 数据源        | 使用场景           |
| :--- | :--------- | :--------- | :------------- |
| 温度   | 设备 > 天气API | 环境监测设备实时数据 | 设备绑定时使用设备数据    |
| 湿度   | 设备 > 天气API | 环境监测设备实时数据 | 设备未绑定时回退到天气API |
| 光照   | 仅设备        | 光照传感器      | 无设备时显示"未监测"    |
| 土壤湿度 | 仅设备        | 土壤湿度传感器    | 无设备时显示"未监测"    |
| 土壤温度 | 仅设备        | 土壤温度传感器    | 无设备时显示"未监测"    |

**数据源切换逻辑**:

```
植物详情页-环境Tab加载
         ↓
    检查是否绑定设备
         ↓
    ┌────┴────┐
   已绑定    未绑定
     ↓         ↓
  读取设备   检查位置信息
  实时数据      ↓
     ↓      ┌──┴──┐
  显示数据  有位置  无位置
              ↓      ↓
           调用天气  显示
             API   "暂无数据"
              ↓
           显示天气
           转换数据
```

**天气API数据转换规则**:

当使用天气API作为数据源时，需要进行以下转换：

| 天气API字段 | 转换规则      | 显示值         |
| :------ | :-------- | :---------- |
| 室外温度    | 室内修正：-2°C | 温度          |
| 室外湿度    | 直接使用      | 湿度          |
| 天气状况    | 转换为光照状态   | 光照：晴朗/多云/阴天 |
| 土壤相关    | 无法获取      | 显示"未监测"     |

**数据状态判定规则**:

根据植物品种的最适环境范围，判定当前数据状态：

| 状态  | 判定条件         | 显示样式 |
| :-- | :----------- | :--- |
| 适宜  | 数据在适宜范围内     | 绿色   |
| 偏低  | 数据低于适宜范围下限   | 蓝色   |
| 偏高  | 数据高于适宜范围上限   | 橙色   |
| 严重  | 数据偏离适宜范围>30% | 红色   |
| 未监测 | 无数据源         | 灰色   |

**验收标准**:

- [ ] 设备绑定时优先显示设备数据
- [ ] 未绑定设备时自动切换到天气API
- [ ] 天气数据正确转换为室内环境参考值
- [ ] 数据状态根据植物品种动态判定
- [ ] 历史趋势图正确展示24小时/7天/30天数据
