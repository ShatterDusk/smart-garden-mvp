**做了个植物医生**

我之前养多肉，叶子上长斑，不知道咋回事。
百度答案乱七八糟，问朋友也是各说各的。

- 拍照 → AI告诉你植物怎么了

## 技术架构（看图说话）

**整体**
```
微信小程序 → Node.js后端 → MySQL数据库
                ↓
          调用AI模型/天气API
```

**分层**
- Controller — 参数校验
- Service — 业务逻辑
- Model — 数据访问

**技术栈**
- 前端：微信小程序
- 后端：Node.js + Express + Sequelize
- 数据库：MySQL
- AI：视觉模型 + 语言模型

## 几个有意思的点

### 双轨会话
- 咨询会话：临时对话，不用绑定植物
- 植物会话：绑定特定植物，有上下文
- 可以"升级"：咨询结果 → 保存为植物档案，历史保留

### 数据补偿
- 设备可能离线
- 设计了补偿机制：天气API数据补充
- 标记数据来源：sensor / weather_api / compensation

### 植物档案记什么
- 档案要记哪些数据，我想了很久
- 后来对着一张多肉AI识别结果反推——它能返回什么数据
- 然后想，要这些数据需要传感器采什么、用户录入什么
- 想清楚这些，档案结构就出来了
