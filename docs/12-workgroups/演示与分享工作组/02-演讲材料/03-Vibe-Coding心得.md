## Prompt很重要

**一开始我以为**：扔句话给AI就行
**结果**：生成的代码乱七八糟

**后来学乖了**：

```
好的Prompt长这样：

【背景】Node.js + Express + Sequelize项目

【任务】实现会话管理，包括：
- 创建会话
- 添加消息
- 组装上下文

【约束】
- 参考UserService的风格
- 用asyncHandler包装错误
- 写JSDoc注释

【输出】
- Service代码
- 单元测试
- 设计决策说明
```

**效果**：代码基本可用，最多小改

## AI代码不等于完美

**常见问题**
- 幻觉：调用不存在的函数
- 过时：用废弃的API
- 边界：不处理异常情况

**所以要**
- ESLint + Prettier 自动检查
- 写测试用例验证
- 人工Review

**数据**：640+测试用例，覆盖率60%，70%是AI生成的骨架

---

## 关键心得

1. **从小任务开始**：别一上来就让AI写整个系统
2. **好Prompt省时间**：花10分钟写Prompt，省2小时调试
3. **保持怀疑**：AI会自信地给错误答案
4. **文档化**：记录有效的Prompt模式
