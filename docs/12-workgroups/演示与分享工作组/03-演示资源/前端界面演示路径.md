# 前端界面演示路径

> 规划演示时的操作流程，确保流畅展示

---

## 演示前准备

### 1. 启动后端服务

```bash
cd backend/server
npm run dev
```

### 2. 启动虚拟设备（可选）

```bash
cd _dev/tools/virtual_device
python app/main.py
```

### 3. 打开微信开发者工具

加载小程序项目: `frontend/`

---

## 演示路径（5分钟）

### 路径1：用户登录 → 植物管理（1分钟）

```
login (登录页)
    ↓ 点击"微信登录"
index (首页)
    ↓ 点击"我的植物"
plants (植物列表)
    ↓ 点击某个植物
plant-detail (植物详情)
```

**演示话术**:
> "用户登录后进入首页，可以看到植物概览。点击进入植物列表，选择一株植物查看详情。"

---

### 路径2：AI诊断功能（2分钟）

```
plant-detail (植物详情)
    ↓ 点击"AI诊断"按钮
quick-analyze (快速分析)
    ↓ 上传植物照片
    ↓ 等待AI分析
    ↓ 查看诊断结果
plant-detail (返回详情，查看诊断卡)
```

**演示话术**:
> "这是核心功能——AI诊断。用户上传植物照片，系统调用智谱GLM进行识别和诊断，生成结构化的诊断卡片。"

---

### 路径3：会话咨询（1分钟）

```
index (首页)
    ↓ 点击"AI助手"
sessions (会话列表)
    ↓ 点击某个会话
qna (对话页面)
    ↓ 输入问题
    ↓ 查看AI回复
```

**演示话术**:
> "用户也可以直接和AI对话，咨询养护问题。会话分为两种：咨询会话不绑定植物，植物会话绑定特定植物，有完整上下文。"

---

### 路径4：设备管理（1分钟）

```
index (首页)
    ↓ 点击"设备管理"
device-manage (设备列表)
    ↓ 点击某个设备
device-detail (设备详情)
    ↓ 查看环境数据
metric-detail (指标详情)
```

**演示话术**:
> "这是设备管理界面。用户可以绑定IoT设备，查看实时环境数据。如果设备离线，系统会用天气API补偿数据。"

## 演示要点

### 首页 (index)

- 展示植物数量、设备数量
- 快捷入口：AI助手、添加植物
- 最近诊断卡片

### 植物详情 (plant-detail)

- 植物基本信息
- 环境数据图表
- 诊断历史
- 养护记录

### AI诊断 (quick-analyze)

- 拍照/选择图片
- 上传进度
- 诊断结果展示
- 健康分数、问题列表、建议

### 设备详情 (device-detail)

- 设备状态（在线/离线）
- 电量显示
- 环境数据趋势图
- 数据来源标记（传感器/天气API）

---

## 演示话术模板

### 开场
> "现在演示一下小程序。首先登录..."

### AI诊断
> "点击AI诊断，上传一张植物照片...系统正在调用智谱GLM进行分析...可以看到诊断结果：健康分数85分，发现叶片有轻微黄化，建议增加光照。"

### 设备管理
> "这是绑定的传感器设备。可以看到温度、湿度、土壤湿度的实时数据。注意这个标记——is_stale=true，说明这条数据是天气API补偿的，因为传感器当时离线。"

### 会话
> "用户也可以直接问AI问题。比如'多肉多久浇一次水'...AI会根据植物档案和环境数据给出建议。"

---

## 注意事项

1. **提前准备测试数据**: 确保有植物、设备、会话数据
2. **检查网络**: AI功能需要调用外部API
3. **准备图片**: 演示AI诊断时需要植物照片
4. **后端日志**: 可以同时打开后端控制台，展示请求日志
