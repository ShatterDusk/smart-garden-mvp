## 关键设计亮点

### 1. 双轨会话设计

```
sessions.type = 'consultation' → 临时咨询，不绑定植物
sessions.type = 'plant' → 绑定特定植物，有完整上下文
```

### 2. 环境数据补偿

```
environment_readings.data_source:
  - sensor: 真实传感器数据
  - weather_api: 天气API数据
  - compensation: 补偿数据

environment_readings.is_stale = true → 标记为补偿数据
```

### 3. 读数任务状态机

```
reading_tasks.sensor_status:
  pending → received (正常上报)
  pending → compensated (超时补偿)
  compensated → received (传感器补传覆盖)
  pending → failed (设备故障)
```

## 演示话术

> "我们设计了14张表。重点看reading_tasks表——它追踪数据采集状态。当传感器离线时，状态从pending变成compensated，用天气数据补偿。等传感器重新上线，补传数据会覆盖补偿数据。"
