## 一、检查概述

### 1.1 检查目的

| 目的 | 说明 |
|:---|:---|
| **保障准确性** | 确保文档内容与代码实现一致，避免误导开源用户 |
| **保障完整性** | 确保关键功能有完整文档覆盖，方便学习复现 |
| **保障时效性** | 确保文档版本/状态反映最新情况，避免使用过时信息 |
| **保障一致性** | 确保文档间、文档与代码间无矛盾，提升项目可信度 |
| **保障可发现性** | 确保文档结构清晰、索引完整，易于查找 |

### 1.2 检查范围


## 二、基础质量检查（每次提交检查）

### 2.1 文件命名规范

| 检查项 | 通过标准 | 检查目的 |
|:---|:---|:---|
| 文件名无空格 | 使用 `-` 或 `_` 分隔 | 避免跨平台兼容性问题 |
| 文件名无中文 | 仅使用英文和符号 | 避免编码问题 |
| 文件名无特殊字符 | 仅包含 `a-z`, `0-9`, `-`, `_` | 避免文件系统问题 |
| 文件名不超过 100 字符 | 长度合理 | 便于管理和显示 |

**检查方法**: `ls -la docs/current/**/*.{md,png,jpg}`

---

### 2.2 文档头部元信息

| 检查项 | 必填字段 | 检查目的 |
|:---|:---|:---|
| Markdown 文档 | 标题、版本、创建日期、状态 | 确保文档可追溯 |
| 表格文档 | 表头清晰、无合并单元格 | 便于解析和维护 |

**检查方法**: 解析文档头部，确保包含必要元信息

---

### 2.3 链接有效性

| 检查项 | 通过标准 | 检查目的 |
|:---|:---|:---|
| 内部链接 | 无 404 | 避免死链影响阅读体验 |
| 外部链接 | 可访问 | 确保引用资源有效 |
| 图片链接 | 可加载 | 确保文档可正常展示 |

**检查方法**: `remark --quiet docs/current/*.md` 或手动抽检
