## 发现问题汇总

### 🔴 SA-9-001: 缺少 .env.example 文件（高危）

**问题描述**:
- 项目没有 `.env.example` 文件
- 只有 `.env.test.example` 用于测试环境
- 新开发者无法快速了解需要配置哪些环境变量

**风险影响**:
- 新成员上手困难
- 容易遗漏关键配置
- 生产环境配置容易出错

**建议修复**:
创建 `backend/server/.env.example` 文件：

```bash
# ============================================
# 生产环境配置模板
# 复制此文件为 .env 并填写实际值
# ============================================

NODE_ENV=production

# 数据库配置（必需）
DB_HOST=your_db_host
DB_PORT=3306
DB_NAME=smart_garden
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_DIALECT=mysql
DB_SSL=true
DB_LOGGING=false

# JWT 配置（必需）
JWT_SECRET=your_strong_jwt_secret_min_32_chars
JWT_EXPIRES_IN=7d

# 微信配置（必需）
WECHAT_APPID=your_wechat_appid
WECHAT_SECRET=your_wechat_secret
WECHAT_ENV_ID=your_wechat_env_id

# COS 配置（必需）
COS_BUCKET=your_cos_bucket
COS_REGION=ap-shanghai
COS_SECRET_ID=your_secret_id
COS_SECRET_KEY=your_secret_key

# AI 配置（至少配置一个）
GLM_API_KEY=your_glm_api_key
OPENAI_API_KEY=your_openai_key

# 天气 API（可选）
WEATHER_API_KEY=your_weather_key

# 日志配置
LOG_LEVEL=info
LOG_STORAGE_MODE=database
LOG_ACCESS_KEY=your_secure_log_key

# 其他
UPLOAD_PATH=./uploads
MAX_FILE_SIZE=5242880
SSL_VERIFY=true
```

**优先级**: 🔴 高
**状态**: ⏳ 待修复

### 🟠 SA-9-002: 日志访问密钥硬编码默认值（中危）

**文件位置**: `middleware/logAuth.js:L11`

**问题描述**:
```javascript
return process.env.LOG_ACCESS_KEY || 'dev-log-key-2024';
```

**风险影响**:
- 如果生产环境未设置 LOG_ACCESS_KEY，使用默认密钥
- 攻击者可能使用默认密钥访问日志接口

**修复建议**:
```javascript
const LOG_ACCESS_KEY = process.env.LOG_ACCESS_KEY;
if (!LOG_ACCESS_KEY) {
  throw new Error('LOG_ACCESS_KEY 环境变量必须设置');
}
return LOG_ACCESS_KEY;
```

**优先级**: 🟠 中
**状态**: ⏳ 待修复

### 🟡 SA-9-003: envValidator 未覆盖所有必需变量（低危）

**文件位置**: `utils/envValidator.js:L8-15`

**问题描述**:
```javascript
const REQUIRED_VARS = [
  'DB_HOST',
  'DB_NAME',
  'DB_USER',
  'DB_PASSWORD',
  'JWT_SECRET',
];
```

**缺失的必需变量**:
- `WECHAT_APPID` / `WECHAT_SECRET`（微信登录必需）
- `COS_BUCKET` / `COS_REGION`（文件上传必需）
- `AI_PROVIDER` 对应的 API Key（AI 服务必需）

**修复建议**:
```javascript
const REQUIRED_VARS = [
  'DB_HOST',
  'DB_NAME',
  'DB_USER',
  'DB_PASSWORD',
  'JWT_SECRET',
  'WECHAT_APPID',
  'WECHAT_SECRET',
  'COS_BUCKET',
  'COS_REGION',
];

// AI 配置至少需要一个
const AI_REQUIRED_VARS = [
  'GLM_API_KEY',
  'OPENAI_API_KEY',
  'WENXIN_API_KEY',
  'QWEN_API_KEY',
  'HUNYUAN_API_KEY',
];

// 检查至少配置了一个 AI
const hasAIKey = AI_REQUIRED_VARS.some(varName => process.env[varName]);
if (!hasAIKey) {
  logger.error('至少需要一个 AI 服务的 API Key');
  return false;
}
```

**优先级**: 🟡 低
**状态**: ⏳ 待优化

## 环境变量使用模式分析

### 模式1: 必需变量（推荐）✅
```javascript
const JWT_SECRET = process.env.JWT_SECRET;
if (!JWT_SECRET) {
  throw new Error('JWT_SECRET 环境变量必须设置');
}
```

### 模式2: 有安全默认值（风险）⚠️
```javascript
// SA-9-002 问题
const LOG_ACCESS_KEY = process.env.LOG_ACCESS_KEY || 'dev-log-key-2024';
```

### 模式3: 有合理默认值（可接受）✅
```javascript
const JWT_EXPIRES_IN = process.env.JWT_EXPIRES_IN || '7d';
const DB_PORT = process.env.DB_PORT || 3306;
```

### 模式4: 可选变量（合理）✅
```javascript
const WEATHER_API_KEY = process.env.WEATHER_API_KEY;
if (WEATHER_API_KEY) {
  // 启用天气功能
}
```
