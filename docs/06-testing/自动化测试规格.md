## 三、测试规范

### 3.1 命名规范

| 层级 | 规范 | 示例 |
|------|------|------|
| **文件命名** | `{name}.test.js` | `userController.test.js` |
| **describe** | 中文描述模块 | `describe('UserController', () => {})` |
| **it** | 中文描述场景 | `it('新用户登录成功', () => {})` |

### 3.2 Mock 规范

```javascript
// 1. 在导入被测模块前设置 Mock
jest.mock('../../../src/services', () => ({
  UserService: jest.fn(() => mockUserService),
}));

// 2. 导入被测模块
const userController = require('../../../src/controllers/userController');

// 3. 每个测试前清理 mocks
beforeEach(() => {
  jest.clearAllMocks();
});
```

### 3.3 测试数据规范

```javascript
// 使用工厂函数创建测试数据
const mockUser = {
  user_id: 'USER_123',
  nickname: '测试用户',
  avatar_url: 'https://example.com/avatar.jpg',
  created_at: new Date(),
};

// 避免使用真实数据
// ❌ 不要这样
const realUser = await User.findOne({ where: { id: 1 } });

// ✅ 应该这样
const mockUser = { user_id: 'USER_TEST', nickname: 'Mock' };
User.findOne = jest.fn().mockResolvedValue(mockUser);
```

## 四、测试类型

### 4.1 单元测试

测试单个函数/方法的逻辑，Mock 所有依赖。

**覆盖模块**:
- Controllers (5个文件, 79个用例)
- Services (6个文件, 95个用例)
- Middleware (4个文件, 48个用例)
- Utils (2个文件, 20个用例)

### 4.2 集成测试

测试 API 端点，使用真实数据库（测试库）。

**覆盖模块**:
- users API (15个用例)
- plants API (14个用例)
- sessions API (16个用例)
- devices API (12个用例)
- error-handling (5个用例)

### 4.3 特色测试

#### 安全性测试 (28个用例)
- SQL 注入防护
- XSS 攻击防护
- 权限绕过防护
- DoS 攻击防护
- 特殊字符处理

#### 边界条件测试 (65个用例)
- 字符串长度边界
- 数字范围边界
- 分页参数边界
- 数组长度边界
- 空值处理

## 七、执行命令

```bash
cd backend/server

# 运行所有测试
npm test

# 仅运行单元测试
npm run test:unit

# 仅运行集成测试
npm run test:integration

# 仅运行 E2E 测试
npm run test:e2e

# 生成覆盖率报告
npm run test:coverage

# 运行单个测试文件
npm run test:unit -- userController.test.js

# 运行匹配名称的测试
npm run test:unit -- --testNamePattern="登录"
```
