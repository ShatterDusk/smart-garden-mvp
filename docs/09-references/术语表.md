## 业务术语

### 用户相关

| 术语 | 英文 | 定义 |
|:---|:---|:---|
| **用户** | User | 使用系统的注册用户，通过微信号授权登录 |
| **游客** | Guest | 未登录的临时用户，功能受限 |
| **用户配置** | UserConfig | 用户的个性化设置，如通知偏好、主题等 |
| **用户设置** | UserSettings | 用户的应用级设置，如隐私选项 |

### 植物相关

| 术语 | 英文 | 定义 |
|:---|:---|:---|
| **植物档案** | Plant | 用户在系统中创建的植物记录，包含品种、昵称、位置等信息 |
| **植物品种** | Species | 植物的具体品种名称，如"绿萝"、"吊兰" |
| **植物分类** | PlantCategory | 植物的分类，如多肉、花卉、绿植、蔬菜、其他 |
| **健康评分** | HealthScore | AI 对植物健康状况的评分，范围 0-100 |
| **养护记录** | CareRecord | 用户对植物进行的养护操作记录，如浇水、施肥、修剪 |
| **诊断卡** | DiagnosisCard | AI 对植物健康状态的结构化诊断结果 |

### 设备相关

| 术语 | 英文 | 定义 |
|:---|:---|:---|
| **IoT设备** | Device | 智能园艺硬件设备，可监测环境数据 |
| **设备绑定** | DeviceBinding | 将设备与植物档案关联的操作 |
| **设备解绑** | DeviceUnbinding | 解除设备与植物档案的关联 |
| **MAC地址** | MACAddress | 设备的唯一硬件标识 |
| **数据上报** | DataReporting | 设备向服务器发送传感器数据 |
| **数据补传** | DataSupplement | 设备补发之前未成功上报的数据 |

### 环境数据相关

| 术语 | 英文 | 定义 |
|:---|:---|:---|
| **环境读数** | EnvironmentReading | 某一时间点的环境数据集合 |
| **环境指标** | EnvironmentMetric | 具体的环境测量项，如温度、湿度 |
| **传感器数据** | SensorData | 设备传感器采集的原始数据 |
| **天气数据** | WeatherData | 从天气API获取的室外环境数据 |
| **补偿数据** | CompensationData | 当传感器数据缺失时，通过算法或天气API估算的数据 |
| **数据新鲜度** | DataFreshness | 数据的时效性，超过1小时视为过期 |
| **读数任务** | ReadingTask | 记录数据采集计划的任务 |

### 会话相关

| 术语 | 英文 | 定义 |
|:---|:---|:---|
| **会话** | Session | 用户与AI的对话上下文 |
| **咨询会话** | ConsultationSession | 不绑定植物档案的临时AI对话 |
| **植物会话** | PlantSession | 绑定特定植物档案的专属对话 |
| **会话升级** | SessionUpgrade | 将咨询会话转换为植物会话的操作 |
| **消息** | Message | 会话中的单条对话内容 |
| **上下文** | Context | AI分析时引用的背景数据 |
| **上下文开关** | ContextSwitch | 控制AI分析时引用数据范围的配置 |
| **已读位置** | ReadPosition | 用户最后阅读的消息位置 |

### AI相关

| 术语 | 英文 | 定义 |
|:---|:---|:---|
| **AI分析** | AIAnalysis | 使用AI模型对植物图片或描述进行健康分析 |
| **普通分析** | NormalAnalysis | 不引用档案数据的AI分析 |
| **深度分析** | DeepAnalysis | 引用植物档案、环境数据、养护记录的AI分析 |
| **置信度** | Confidence | AI对诊断结果的确定程度，范围 0-1 |
| **问题列表** | Issues | AI识别的植物健康问题 |
| **建议列表** | Suggestions | AI给出的养护建议 |
| **GLM** | - | 智谱AI的大语言模型 |
