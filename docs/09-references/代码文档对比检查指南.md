# 代码与文档对比检查指南

> 本文档提供系统化的方法来检查代码实现与设计文档之间的差异。

---

## 一、检查流程

### 1.1 后端检查清单

#### 1.1.1 模型 vs 数据库设计

```bash
# 检查步骤
1. 读取 backend/server/src/models/*.js
2. 对比 02-数据库设计.md 中的表结构
3. 检查字段名、类型、约束是否一致
```

**检查要点**：

| 检查项 | 代码位置 | 文档位置 | 常见问题 |
|:---|:---|:---|:---|
| 字段名 | `models/*.js` | 02-数据库设计.md | 命名不一致（如 last_login_at vs last_login_time） |
| 字段类型 | `DataTypes.XXX` | 表结构定义 | 类型不匹配 |
| 默认值 | `defaultValue` | 默认值列 | 默认值不一致 |
| 约束 | `allowNull`, `unique` | 约束说明 | 约束缺失 |
| 索引 | `indexes` | 索引设计章节 | 索引不一致 |
| 关联关系 | `references` | 外键设计章节 | 外键缺失 |
| getter方法 | `getterMethods` | 通常不文档化 | 新增方法未记录 |

#### 1.1.2 路由 vs API接口设计

```bash
# 检查步骤
1. 读取 backend/server/src/routes/*.js
2. 对比 03-API接口设计.md
3. 检查路径、方法、参数是否一致
```

**检查要点**：

| 检查项 | 代码位置 | 文档位置 | 常见问题 |
|:---|:---|:---|:---|
| API路径 | `router.Xxx()` | 接口路径列 | 路径不一致（如 /users/login vs /auth/login） |
| HTTP方法 | `router.get/post/put/delete` | 方法列 | 方法不匹配 |
| 参数定义 | 路由处理函数 | 请求参数表 | 参数缺失或类型不一致 |
| 响应格式 | controller返回值 | 响应示例 | 字段命名不一致 |
| 废弃接口 | 代码中标记废弃 | 文档中标记废弃 | 状态不一致 |

### 1.2 前端检查清单

#### 1.2.1 页面 vs 前端页面设计

```bash
# 检查步骤
1. 检查 pages/ 目录下的页面文件
2. 对比 05-前端页面设计.md
3. 检查页面结构、交互是否一致
```

**检查要点**：

| 检查项 | 代码位置 | 文档位置 | 常见问题 |
|:---|:---|:---|:---|
| 页面路径 | `pages/xxx/xxx` | 页面路径列 | 路径不一致 |
| 页面状态 | `data: {}` | 页面状态表 | 状态字段缺失 |
| 事件处理 | `bind:xxx` | 事件处理表 | 事件未文档化 |
| 组件使用 | `<component />` | 组件设计章节 | 组件未文档化 |

#### 1.2.2 API调用 vs API接口设计

```bash
# 检查步骤
1. 读取 utils/api.js
2. 对比 03-API接口设计.md
3. 检查调用的API是否在文档中
```

## 二、当前已发现的差异

### 2.1 API路径差异（需统一）

| 实际代码路径 | 文档路径 | 建议统一为 |
|:---|:---|:---|
| `/users/login` | `/auth/login` | `/users/login`（以代码为准） |
| `/users/guest-login` | `/auth/guest` | `/users/guest-login` |
| `/users/profile` | `/users/me` | `/users/profile` |
| `/users/settings` | `/users/config` | `/users/settings` |

### 2.2 文档缺失的API

| API | 方法 | 路径 | 说明 |
|:---|:---:|:---|:---|

| 设置用户配置项 | POST | `/users/config` | 设置单个配置项 |

## 三、自动化检查脚本（建议）

### 3.1 后端API检查脚本

```javascript
// scripts/check-api.js
const fs = require('fs');
const path = require('path');

// 读取所有路由文件
const routesDir = path.join(__dirname, '../backend/server/src/routes');
const routeFiles = fs.readdirSync(routesDir).filter(f => f.endsWith('.js'));

const apis = [];

routeFiles.forEach(file => {
  const content = fs.readFileSync(path.join(routesDir, file), 'utf8');
  
  // 提取路由定义
  const routeMatches = content.match(/router\.(get|post|put|delete)\(['"]([^'"]+)['"]/g);
  
  if (routeMatches) {
    routeMatches.forEach(match => {
      const [, method, path] = match.match(/router\.(\w+)\(['"]([^'"]+)['"]/);
      apis.push({
        file: file,
        method: method.toUpperCase(),
        path: path
      });
    });
  }
});

console.log('发现的API列表：');
console.table(apis);
```

### 3.2 模型字段检查脚本

```javascript
// scripts/check-models.js
const fs = require('fs');
const path = require('path');

const modelsDir = path.join(__dirname, '../backend/server/src/models');
const modelFiles = fs.readdirSync(modelsDir).filter(f => f.endsWith('.js') && f !== 'index.js');

const models = {};

modelFiles.forEach(file => {
  const modelName = file.replace('.js', '');
  const content = fs.readFileSync(path.join(modelsDir, file), 'utf8');
  
  // 提取字段定义
  const fieldMatches = content.match(/(\w+):\s*\{[\s\S]*?type:\s*DataTypes\.(\w+)/g);
  
  if (fieldMatches) {
    models[modelName] = fieldMatches.map(match => {
      const [, fieldName] = match.match(/(\w+):\s*\{/);
      const [, fieldType] = match.match(/type:\s*DataTypes\.(\w+)/);
      return { name: fieldName, type: fieldType };
    });
  }
});

console.log('模型字段汇总：');
Object.entries(models).forEach(([name, fields]) => {
  console.log(`\n${name}:`);
  console.table(fields);
});
```

## 四、定期维护建议

### 4.1 每次迭代前

1. **检查新增代码**：查看上次文档更新后新增的代码
2. **对比文档**：将新增功能与文档对比
3. **更新文档**：在开发前更新设计文档

### 4.2 每次迭代后

1. **代码审查**：审查时同步检查文档一致性
2. **文档更新**：将代码变更同步到文档
3. **版本标记**：在文档中标记实现状态

### 4.3 推荐工具

| 工具类型 | 推荐工具 | 用途 |
|:---|:---|:---|
| API文档生成 | Swagger/OpenAPI | 从代码生成API文档 |
| 数据库文档 | dbdocs.io | 从数据库生成ER图 |
| 代码注释 | JSDoc | 标准化代码注释 |
| 版本对比 | Git diff | 对比代码变更 |
