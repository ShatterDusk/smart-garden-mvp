# 文档管理规范 - 项目文档组织最佳实践

## 摘要

本文档定义了 proj-alpha 项目的文档管理规范，包括目录结构、文件命名、文档模板、版本管理和状态流转等。通过统一的规范，确保项目文档的可维护性和可查找性。

## 目录结构规范

### 标准目录结构

```
docs/
├── README.md                     # 文档首页
├── 01-product/                   # 产品文档
├── 02-architecture/              # 架构设计
├── 03-frontend/                  # 前端文档
├── 04-backend/                   # 后端文档
├── 05-process/                   # 业务流程
├── 06-testing/                   # 测试文档
├── 07-operations/                # 运维文档
├── 08-tasks/                     # 任务管理
├── 09-references/                # 参考资料
└── 10-project/                   # 项目管理
```

### 目录命名规则

- 使用 **数字前缀**（01-10）表示优先级/顺序
- 使用 **英文小写** 作为目录名
- 子目录不使用数字前缀

## 文件命名规范

### 文档类文件

```
{序号}-{描述}.md

示例：
01-需求规格说明书.md
02-数据库设计.md
```

### 规格类文件

```
{日期}-{功能}-spec.md
{日期}-{功能}-tasks.md
{日期}-{功能}-checklist.md

示例：
2026-04-07-环境数据-spec.md
2026-04-07-环境数据-tasks.md
```

### README 文件

每个目录必须包含 `README.md` 作为索引。

## 文档模板规范

### 标准文档头部

每个文档必须包含以下元信息：

```markdown
# 文档标题

## 元信息
- **文档类型**: 需求/设计/流程/任务/参考
- **版本**: V1.0
- **创建日期**: 2026-04-07
- **最后更新**: 2026-04-07
- **作者**: [姓名]
- **状态**: 草稿/评审中/已发布/已归档
- **关联文档**: [链接]

---

## 正文内容...
```

### 文档类型说明

| 类型 | 说明 | 存放位置 |
|:---|:---|:---|
| 需求 | 产品需求、用户故事 | 01-product/ |
| 设计 | 架构、数据库、API设计 | 02-architecture/ |
| 流程 | 业务流程说明 | 05-process/ |
| 任务 | 开发任务、TODO | 08-tasks/ |
| 参考 | 术语表、检查清单 | 09-references/ |

## 版本管理规范

### 版本号规则

| 版本号 | 含义 | 更新时机 |
|:---|:---|:---|
| V0.x | 草稿 | 初稿编写阶段 |
| V1.0 | 首次发布 | 评审通过后 |
| V1.x | 小修订 | 内容修正、补充 |
| V2.0 | 重大更新 | 结构重构、需求变更 |

### 版本变更记录

在文档末尾添加变更记录：

```markdown
---

## 变更记录

| 日期 | 版本 | 变更内容 | 作者 |
|:---|:---:|:---|:---|
| 2026-04-07 | V1.0 | 初始版本 | [姓名] |
| 2026-04-08 | V1.1 | 补充XX内容 | [姓名] |
```

## 文档状态流转

```
草稿 → 评审中 → 已发布 → 已归档
  ↓       ↓        ↓
  └──── 废弃 ──────┘
```

| 状态 | 说明 |
|:---|:---|
| 草稿 | 正在编写，内容不完整 |
| 评审中 | 已完成初稿，等待评审 |
| 已发布 | 评审通过，正式生效 |
| 已归档 | 已过时，仅作参考 |
| 废弃 | 不再使用 |

## 目录 README 模板

每个目录的 README.md 必须包含：

```markdown
# 目录名

## 目录说明
简要说明本目录存放什么类型的文档。

## 文档清单
| 序号 | 文档名 | 版本 | 状态 | 最后更新 |
|:---:|:---|:---:|:---:|:---:|
| 01 | [文档名](./文档.md) | V1.0 | ✅ 已发布 | 2026-04-07 |

## 快速导航
- [返回上级](../README.md)
- [文档首页](../../README.md)
```

## 文档创建流程

### 新建文档步骤

1. **确定位置** - 根据文档类型选择正确的目录
2. **命名文件** - 按命名规范命名
3. **复制模板** - 使用标准文档头部模板
4. **填写元信息** - 完整填写所有元信息字段
5. **更新索引** - 更新所在目录的 README.md
6. **提交评审** - 完成后提交评审

### 文档更新步骤

1. **更新内容** - 修改文档内容
2. **更新元信息** - 更新"最后更新"日期
3. **更新版本** - 根据变更程度更新版本号
4. **添加变更记录** - 在变更记录表中添加一行
5. **更新索引** - 如有必要，更新目录 README.md

## 禁止事项

1. ❌ 不要在文档根目录外随意创建文档
2. ❌ 不要使用中文目录名（除 docs 外）
3. ❌ 不要遗漏文档头部元信息
4. ❌ 不要手动修改已归档文档
5. ❌ 不要保留多个版本的相同文档（应使用版本控制）

## 双区制管理

推荐采用 **current/archive** 双区制：

```
docs/
├── README.md              # 文档总入口
├── current/               # 当前工作区（活跃文档）
│   ├── 01-product/
│   ├── 02-architecture/
│   └── ...
└── archive/               # 历史归档区（只读备份）
    ├── 2026-04-07-docs备份/
    └── ...
```

### 优势

- **current/** - 存放当前生效的文档，便于日常查阅
- **archive/** - 存放历史版本，便于追溯和回滚
- 避免文档散落在多个位置
- 清晰的文档生命周期管理
