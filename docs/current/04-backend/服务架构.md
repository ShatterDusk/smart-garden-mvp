## 服务概述

后端服务基于 **Node.js + Express** 构建，采用分层架构设计，提供 RESTful API 供微信小程序和 IoT 设备调用。

### 核心特性

| 特性 | 说明 |
|:---|:---|
| **分层架构** | Controller → Service → Model 三层分离 |
| **领域驱动** | 按业务域组织代码（用户域、植物域、设备域等） |
| **统一响应** | 标准响应格式 `{ code, message, data }` |
| **多重认证** | JWT 用户认证 + DeviceId 设备认证 |
| **定时任务** | 天气数据同步、数据补偿等定时任务 |
| **日志追踪** | 结构化日志 + 前端日志收集 |

## 服务层架构

### 基础服务类 (BaseService)

所有领域服务继承 `BaseService`，提供通用 CRUD 方法：

```javascript
class BaseService {
  async findById(id)           // 按主键查询
  async findOne(conditions)    // 单条查询
  async findAll(conditions, options)  // 列表查询
  async create(data)           // 创建记录
  async update(id, data)       // 更新记录
  async delete(id)             // 删除记录
  async paginate(conditions, page, pageSize, options)  // 分页查询
}
```

### 领域服务

| 服务 | 职责 | 依赖 |
|:---|:---|:---|
| **UserService** | 用户管理、认证、配置 | BaseService |
| **PlantService** | 植物档案 CRUD、诊断卡查询 | BaseService |
| **SessionService** | 会话管理、消息处理、诊断卡创建 | BaseService |
| **DeviceService** | 设备绑定、数据上报 | BaseService, weatherService |
| **EnvironmentService** | 环境数据查询、补偿数据 | BaseService, compensationService |
| **CareRecordService** | 养护记录管理 | BaseService |
| **aiService** | AI 分析、GLM API 调用 | - |
| **weatherService** | 天气数据获取 | - |
| **compensationService** | 数据补偿计算 | - |

## 认证机制

### JWT 用户认证

```javascript
// 请求头
Authorization: Bearer <jwt_token>

// 适用于
- 微信小程序所有 API 调用
- 用户相关操作
```

### 设备认证

```javascript
// 请求体
{
  "deviceId": "DEV_xxx",
  "metrics": { ... }
}

// 适用于
- IoT 设备数据上报
- /api/devices/data
```

## API 响应规范

### 成功响应

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

### 错误响应

```json
{
  "code": 400,
  "message": "请求参数错误",
  "data": null
}
```

### 错误码定义

| 错误码 | 说明 |
|:---:|:---|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
| 1001-1010 | 业务错误码 |

## 开发指南

### 启动开发服务器

```bash
cd backend/server
npm install
npm run dev
```

