## 环境配置

### 1. 安装 Node.js

```bash
# 使用 NVM 安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 18
nvm use 18
node -v  # v18.x.x
```

### 2. 安装 MySQL

```bash
# Ubuntu
sudo apt update
sudo apt install mysql-server-8.0
sudo mysql_secure_installation

# 创建数据库
sudo mysql -u root -p
CREATE DATABASE smart_garden CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'garden_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON smart_garden.* TO 'garden_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 3. 安装 Nginx

```bash
# Ubuntu
sudo apt update
sudo apt install nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

### 4. 安装 PM2

```bash
npm install -g pm2
pm2 --version
```

## 应用部署

### 1. 代码部署

```bash
# 克隆代码
cd /var/www
git clone https://github.com/ShatterDusk/smart-garden-mvp.git
cd smart-garden-mvp/backend/server

# 安装依赖
npm install --production
```

### 2. 环境变量配置

创建 `.env` 文件：

```bash
cd /var/www/smart-garden-mvp/backend/server
cp .env.example .env
nano .env
```

配置内容：

```env
# 服务器配置
NODE_ENV=production
PORT=3000
HOST=0.0.0.0

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=smart_garden
DB_USER=garden_user
DB_PASSWORD=your_password
DB_DIALECT=mysql

# JWT 配置
JWT_SECRET=your_jwt_secret_key_here
JWT_EXPIRES_IN=7d

# 微信小程序配置
WECHAT_APPID=your_wechat_appid
WECHAT_SECRET=your_wechat_secret

# AI 服务配置（GLM/OpenAI）
AI_PROVIDER=glm
AI_API_KEY=your_ai_api_key
AI_MODEL=glm-4

# 腾讯云 COS 配置
COS_SECRET_ID=your_cos_secret_id
COS_SECRET_KEY=your_cos_secret_key
COS_BUCKET=your_bucket_name
COS_REGION=ap-guangzhou
COS_DOMAIN=https://your-cos-domain.com

# 天气 API 配置
WEATHER_API_KEY=your_weather_api_key

# 日志配置
LOG_LEVEL=info
LOG_STORAGE_MODE=database
LOG_ACCESS_KEY=your_log_access_key
```

### 3. 数据库迁移

```bash
cd /var/www/smart-garden-mvp/backend/server

# 执行 Sequelize 迁移
npx sequelize-cli db:migrate

# 初始化基础数据（如果有）
npx sequelize-cli db:seed:all
```

### 4. PM2 启动应用

```bash
cd /var/www/smart-garden-mvp/backend/server

# 启动应用
pm2 start server.js --name "smart-garden-server"

# 查看状态
pm2 status
pm2 logs smart-garden-server

# 保存 PM2 配置
pm2 save
pm2 startup
```

## Nginx 配置

### 反向代理配置

创建 Nginx 配置文件：

```bash
sudo nano /etc/nginx/sites-available/smart-garden
```

配置内容：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 日志配置
    access_log /var/log/nginx/smart-garden-access.log;
    error_log /var/log/nginx/smart-garden-error.log;

    # 文件上传大小限制
    client_max_body_size 10M;

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # 健康检查
    location /health {
        proxy_pass http://127.0.0.1:3000/health;
        access_log off;
    }

    # 静态文件（如果有）
    location /static/ {
        alias /var/www/smart-garden-mvp/backend/server/public/;
        expires 30d;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/smart-garden /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 日志管理

```bash

tail -f /var/www/smart-garden-mvp/backend/server/logs/app.log

# 查看错误日志
tail -f /var/www/smart-garden-mvp/backend/server/logs/error.log

# 日志轮转（使用 logrotate）
sudo nano /etc/logrotate.d/smart-garden
```

logrotate 配置：

```
/var/www/smart-garden-mvp/backend/server/logs/*.log {
    daily
    rotate 14
    compress
    delaycompress
    missingok
    notifempty
    create 0644 www-data www-data
    sharedscripts
    postrotate
        pm2 reload smart-garden-server
    endscript
}
```

## 备份策略

### 数据库备份

```bash
# 创建备份脚本
sudo nano /opt/backup/backup-db.sh
```

脚本内容：

```bash
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/opt/backup/database"
DB_NAME="smart_garden"
DB_USER="garden_user"
DB_PASS="your_password"

mkdir -p $BACKUP_DIR
mysqldump -u $DB_USER -p$DB_PASS $DB_NAME | gzip > $BACKUP_DIR/${DB_NAME}_${DATE}.sql.gz

# 保留最近 30 天的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete
```

设置定时任务：

```bash
chmod +x /opt/backup/backup-db.sh
sudo crontab -e

# 每天凌晨 2 点备份
0 2 * * * /opt/backup/backup-db.sh
```

## 故障排查

### 常见问题

| 问题 | 可能原因 | 解决方案 |
|:---|:---|:---|
| 应用无法启动 | 端口被占用 | `lsof -i :3000` 查看占用进程 |
| 数据库连接失败 | 配置错误或权限问题 | 检查 `.env` 配置和 MySQL 用户权限 |
| 502 Bad Gateway | 后端服务未启动 | `pm2 status` 检查服务状态 |
| 文件上传失败 | 文件大小限制 | 检查 Nginx `client_max_body_size` |
| 内存不足 | 内存泄漏或配置不当 | 增加服务器内存或优化代码 |

### 诊断命令

```bash
# 检查服务状态
pm2 status
systemctl status nginx
systemctl status mysql

# 检查端口监听
netstat -tlnp | grep :3000

# 检查资源使用
free -h
df -h
top

# 查看最近错误
pm2 logs smart-garden-server --lines 50
```
