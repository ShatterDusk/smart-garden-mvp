## 问题描述

`environment_metrics` 表（环境指标定义表）没有初始化数据，导致环境数据查询失败。

### 影响
- `environment_reading_values` 表外键关联 `environment_metrics.metric_code`
- 没有指标定义数据，环境数据无法正常显示

---

## 解决方案

执行 SQL 插入环境指标定义数据。

### SQL 文件位置

`DataBase/init-environment-metrics.sql`

## 特殊数据处理（通过 unit 字段识别）

前端根据 `unit` 字段值，以不同方式处理数据：

| unit 值 | 数据类型 | 处理方式 | 示例指标 |
|:---:|:---|:---|:---|
| `code` | 编码型 | 前端映射到文本和图标 | weather_condition (100→晴 ☀️) |
| `°` | 角度型 | 前端映射到方位文字 | wind_direction_360 (123°→东南风) |
| `°C`, `%`, `hPa`... | 标准单位 | 正常数值展示 | temperature, humidity |

### 前端处理示例

```javascript
// 根据 unit 判断数据类型
const metric = await getMetricInfo(metricCode);

switch (metric.unit) {
  case 'code':
    // 编码型：查表映射
    displayValue = weatherCodeMap[value].text + weatherCodeMap[value].icon;
    break;
  case '°':
    // 角度型：映射到方位
    displayValue = degreeToDirection(value); // 123° → 东南风
    break;
  default:
    // 标准单位：直接展示
    displayValue = value + metric.unit;
}
```
