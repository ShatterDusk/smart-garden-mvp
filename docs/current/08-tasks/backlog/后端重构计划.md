## 一、重构背景

### 1.1 当前问题

| 问题 | 描述 | 影响 |
|:---|:---|:---|
| 缺少Service层 | Controller直接操作Model，业务逻辑混杂 | 代码复用性差，测试困难 |
| 路由结构扁平 | 11个路由文件无领域分组 | 维护成本高，职责不清 |
| 模块边界模糊 | 设备模块与环境模块职责交叉 | 代码耦合，扩展困难 |
| 响应格式不统一 | 部分接口返回格式不一致 | 前端处理复杂 |

### 1.2 设计目标

根据 `01-系统架构设计.md` V3.0 的要求：

```
Controller → Service → Model
```

- Controller: 只负责请求/响应处理
- Service: 封装业务逻辑
- Model: 数据访问层

## 七、附录

### A. Service层设计规范

```javascript
// services/BaseService.js
class BaseService {
  constructor(model) {
    this.model = model;
  }

  async findById(id) {
    return this.model.findByPk(id);
  }

  async findOne(conditions) {
    return this.model.findOne({ where: conditions });
  }

  async findAll(conditions = {}) {
    return this.model.findAll({ where: conditions });
  }

  async create(data) {
    return this.model.create(data);
  }

  async update(id, data) {
    const record = await this.findById(id);
    if (!record) return null;
    return record.update(data);
  }

  async delete(id) {
    const record = await this.findById(id);
    if (!record) return null;
    return record.destroy();
  }
}

module.exports = BaseService;
```

### B. Controller层设计规范

```javascript
// controllers/xxxController.js
const XxxService = require('../services/XxxService');
const { success, error } = require('../utils/response');

const getXxx = async (req, res) => {
  try {
    const { id } = req.params;
    const result = await XxxService.getById(id);
    if (!result) {
      return error(res, '记录不存在', 404);
    }
    return success(res, result);
  } catch (err) {
    return error(res, err.message, 500);
  }
};

module.exports = { getXxx };
```
