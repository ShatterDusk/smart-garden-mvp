## 1. 系统定位

AI 养护任务系统是智能园艺助手的核心功能模块，通过 AI 分析植物健康状态，自动生成个性化养护任务，帮助用户科学养护植物。

### 1.1 解决的问题

| 问题 | 解决方案 |
|:---|:---|
| 不知道何时浇水施肥 | AI 根据环境数据和植物状态智能推荐 |
| 养护计划混乱 | 统一任务管理，今日待办清晰展示 |
| 忘记养护 | 推送提醒 + 角标提示 |
| 养护记录零散 | 完成任务自动转为养护记录 |

### 1.2 核心价值

- **智能化**: 基于 AI 诊断自动生成任务
- **个性化**: 针对不同植物品种定制养护计划
- **闭环管理**: 任务 → 执行 → 记录 → 反馈

## 3. 核心概念

### 3.1 CareTask（养护任务）

待执行的养护任务，包含计划时间、优先级、AI推荐理由等。

```javascript
{
  taskId: 'TASK_001',
  type: 'water',
  title: '浇水',
  status: 'pending',  // pending | completed | skipped
  source: 'ai',       // ai | rule | manual
  scheduledDate: '2026-04-12',
  aiReason: '环境湿度低于30%'
}
```

### 3.2 CareRecord（养护记录）

已完成的养护操作，作为历史记录保存。

```javascript
{
  recordId: 'RECORD_001',
  actionType: 'water',
  description: '已完成浇水',
  performedAt: '2026-04-12T09:00:00Z',
  source: 'task',     // 标记来源是任务
  taskId: 'TASK_001'  // 关联原任务
}
```

### 3.3 任务来源

| 来源 | 说明 | 示例 |
|:---|:---|:---|
| **AI** | 基于诊断卡智能生成 | 健康评分低时生成紧急养护任务 |
| **Rule** | 基于规则自动生成 | 每周一生成浇水任务 |
| **Manual** | 用户手动创建 | 用户自定义提醒 |

## 5. 与现有系统的集成

### 5.1 与诊断卡系统集成

```
AI 诊断 ──► 生成诊断卡 ──► 解析 issues/suggestions ──► 生成 CareTask
```

### 5.2 与养护记录系统集成

```
CareTask (pending) ──► 点击完成 ──► 填写执行信息 ──► 生成 CareRecord
```

### 5.3 与植物详情页集成

```
植物详情页 ──► 养护 Tab ──► 今日待办 + 历史记录
```
