## 2. 数据上传流程架构

```mermaid
sequenceDiagram
    participant Sensor as IoT传感器
    participant API as POST /api/devices/data
    participant Auth as deviceAuthMiddleware
    participant Controller as deviceController
    participant DeviceSvc as DeviceService
    participant EnvSvc as EnvironmentService
    participant CompSvc as compensationService
    participant DB as MySQL数据库
    
    Sensor->>API: 上报环境数据
    API->>Auth: 设备认证
    Auth->>Auth: 验证设备ID和密钥
    
    alt 认证失败
        Auth-->>Sensor: 401 Unauthorized
    else 认证成功
        Auth->>Controller: reportData()
        Controller->>DeviceSvc: reportDeviceData(data)
        
        DeviceSvc->>DB: 查询设备信息
        DB-->>DeviceSvc: 设备信息
        
        alt 设备不存在
            DeviceSvc-->>Sensor: 404 设备不存在
        else 设备存在
            DeviceSvc->>DB: 查询绑定植物
            DB-->>DeviceSvc: 植物信息
            
            alt 植物不存在
                DeviceSvc-->>Sensor: 404 植物不存在
            else 植物存在
                DeviceSvc->>EnvSvc: processDeviceEnvironmentData()
                
                EnvSvc->>DB: 查询ReadingTask
                DB-->>EnvSvc: Task状态
                
                alt Task不存在
                    EnvSvc->>DB: 创建新Task
                end
                
                alt Task.sensor_status = pending
                    EnvSvc->>CompSvc: createSensorReading()
                    CompSvc->>DB: 创建Reading
                    CompSvc->>DB: 创建Values
                    CompSvc->>DB: 更新Task状态
                else Task.sensor_status = compensated
                    EnvSvc->>CompSvc: coverCompensatedData()
                    CompSvc->>DB: 删除补偿数据
                    CompSvc->>DB: 创建真实数据
                    CompSvc->>DB: 更新Task状态
                else Task.sensor_status = received
                    alt isSupplement = true
                        EnvSvc-->>Sensor: 409 已有真实数据
                    else
                        EnvSvc-->>Sensor: 200 数据已存在
                    end
                end
                
                EnvSvc-->>DeviceSvc: 返回结果
                DeviceSvc->>DB: 更新设备状态
                DeviceSvc-->>Sensor: 200 数据上报成功
            end
        end
    end
```

## 5. 定时任务执行流程

```mermaid
sequenceDiagram
    participant Cron as Cron定时器
    participant Job as environmentSyncJob
    participant Task as ReadingTask
    participant Weather as weatherService
    participant Comp as compensationService
    participant DB as MySQL
    
    Note over Cron: 每2小时整点执行
    Cron->>Job: 触发同步任务
    
    rect rgb(200, 220, 240)
        Note over Job,DB: 阶段1: 生成任务
        Job->>DB: 查询所有植物
        DB-->>Job: 植物列表
        
        loop 每个植物
            Job->>Task: 创建ReadingTask
            Task->>DB: 插入task记录
            Note right of Task: sensor_status='pending'<br/>weather_status='pending'
        end
    end
    
    rect rgb(220, 240, 200)
        Note over Job,DB: 阶段2: 获取天气
        loop 每个植物
            Job->>Weather: getWeatherForPlant()
            Weather->>Weather: 调用天气API
            Weather-->>Job: 天气数据
            Job->>DB: 创建weather_api reading
            Job->>Task: 更新weather_status='received'
        end
    end
    
    Note over Cron: 整点后5分钟执行
    Cron->>Job: 触发补偿检查
    
    rect rgb(240, 220, 200)
        Note over Job,DB: 阶段3: 补偿检查
        Job->>Comp: checkAndCompensateAll()
        Comp->>DB: 查询超时pending task
        
        loop 每个超时task
            alt 有上期数据
                Comp->>DB: 查询上期sensor reading
                Comp->>DB: 创建compensated reading
                Note right of Comp: is_stale=true
                Comp->>Task: 更新sensor_status='compensated'
            else 无上期数据
                Comp->>Task: 更新sensor_status='failed'
            end
        end
    end
```

## 6. 补偿数据覆盖流程

```mermaid
sequenceDiagram
    participant Sensor as IoT传感器<br/>(恢复连接)
    participant API as POST /api/devices/data
    participant EnvSvc as EnvironmentService
    participant CompSvc as compensationService
    participant Task as ReadingTask
    participant DB as MySQL
    
    Note over Sensor,DB: 传感器离线期间<br/>已生成补偿数据
    
    Sensor->>API: 补传数据<br/>isSupplement=true
    API->>EnvSvc: processDeviceEnvironmentData()
    
    EnvSvc->>Task: 查询task状态
    Task-->>EnvSvc: sensor_status='compensated'
    
    EnvSvc->>CompSvc: coverCompensatedData()
    
    CompSvc->>DB: 开启事务
    
    CompSvc->>DB: 查询补偿reading<br/>is_stale=true
    DB-->>CompSvc: 补偿reading
    
    CompSvc->>DB: 删除reading_values
    CompSvc->>DB: 删除reading
    
    CompSvc->>DB: 创建真实reading<br/>is_stale=false
    CompSvc->>DB: 创建真实values
    
    CompSvc->>Task: 更新sensor_status='received'
    
    CompSvc->>DB: 提交事务
    
    EnvSvc-->>Sensor: 200 数据上报成功
    
    Note over Sensor: 补传成功<br/>补偿数据已被真实数据覆盖
```

## 架构设计原则

### 1. 分层架构

- **客户端层**：微信小程序、IoT设备
- **API网关层**：路由分发、认证授权
- **控制器层**：请求处理、参数验证
- **服务层**：业务逻辑、数据聚合
- **数据访问层**：ORM映射、数据库操作

### 2. 服务职责划分

| 服务 | 职责 | 依赖 |
|------|------|------|
| PlantService | 植物管理、数据聚合 | EnvironmentService |
| DeviceService | 设备管理、数据上报 | EnvironmentService |
| EnvironmentService | 环境数据处理、查询 | compensationService、weatherService |
| compensationService | 数据补偿、覆盖 | 无 |
| weatherService | 天气API调用 | 无 |

### 3. 数据流向

```
IoT设备 → DeviceService → EnvironmentService → compensationService → MySQL
定时任务 → EnvironmentService → weatherService → MySQL
前端页面 → API → Service → MySQL
```

### 4. 扩展性设计

- **服务解耦**：服务间单向依赖，无循环依赖
- **接口统一**：统一的API响应格式
- **错误处理**：统一的错误处理中间件
- **日志记录**：完整的日志记录体系
