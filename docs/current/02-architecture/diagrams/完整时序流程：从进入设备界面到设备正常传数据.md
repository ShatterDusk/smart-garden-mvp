## 完整时序流程：从进入设备界面到设备正常传数据

```mermaid
sequenceDiagram
    participant U as 用户
    participant C as 小程序
    participant D as ESP32设备
    participant S as 后端
    participant DB as 数据库

    Note over U,C: ===== Phase 1: 进入设备管理界面 =====
    U->>C: 从植物详情页点击"设备管理"
    C->>C: 携带 plantId 进入设备管理页
    
    Note over C,S: ===== Phase 2: 加载页面数据 =====
    C->>S: GET /api/plants/{plantId}<br/>Authorization: Bearer {token}
    S->>DB: 查询植物信息
    DB-->>S: { plantId, currentDevice: {...} }
    S-->>C: 植物详情（含当前绑定设备）
    
    C->>S: GET /api/devices<br/>Authorization: Bearer {token}
    S->>DB: 查询用户设备列表
    DB-->>S: [{ deviceId, macAddress, status, ... }]
    S-->>C: 设备列表
    
    Note over C,C: 页面显示：<br/>1. 当前绑定设备（如有）<br/>2. 可用设备列表

    Note over U,D: ===== Phase 3: 设备配网 =====
    U->>D: 长按设备按键3秒，进入配网模式
    D->>D: 开启 AP 模式 (proj-alpha-XXXX)
    U->>C: 连接设备 WiFi
    U->>C: 输入家庭 WiFi 信息
    C->>D: SmartConfig 广播配置
    D->>D: 连接家庭 WiFi
    D->>D: 广播设备信息 (UDP/mDNS)<br/>{ macAddress, deviceName }
    
    Note over C,C: ===== Phase 4: 发现设备 =====
    C->>C: UDP/mDNS 扫描局域网<br/>(MVP版本暂用后端接口)
    C->>C: 发现设备列表（本地）<br/>[{ macAddress, deviceName }]
    
    Note over U,S: ===== Phase 5: 绑定设备 =====
    U->>C: 选择设备
    C->>S: POST /api/devices/bind<br/>Authorization: Bearer {token}<br/>{ macAddress, deviceName, plantId }
    
    S->>S: userId = 从 token 解析
    S->>DB: 查询设备 WHERE mac_address = ?
    
    alt 设备不存在
        S->>S: 生成 deviceId = DEVICE_xxx
        S->>DB: INSERT devices<br/>(device_id, user_id, mac_address, status='online')
    else 设备已存在
        S->>DB: 更新设备状态
    end
    
    S->>DB: UPDATE plants<br/>SET current_device_id = ?
    S-->>C: { deviceId, macAddress, status: "online" }
    C-->>U: 绑定成功
    
    Note over D,S: ===== Phase 6: 设备数据上报 =====
    loop 定期上报（如每60秒）
        D->>S: POST /api/devices/data<br/>{ macAddress, timestamp, metrics }
        S->>DB: 查询设备 WHERE mac_address = ?
        DB-->>S: { deviceId, user_id }
        S->>DB: 反向查询 Plant<br/>WHERE current_device_id = ?
        DB-->>S: { plantId }
        S->>DB: 创建 EnvironmentReading
        S->>DB: 更新 device.last_heartbeat
        S-->>D: { readingId }
    end
```

## 各阶段详细说明

### Phase 1-2: 进入界面加载数据

| 步骤 | API | 说明 |
|:---|:---|:---|
| 1 | `GET /api/plants/{plantId}` | 获取植物详情，包含 `currentDevice` |
| 2 | `GET /api/devices` | 获取用户已绑定的设备列表 |

### Phase 3: 设备配网

| 步骤 | 说明 |
|:---|:---|
| 1 | 设备进入配网模式（AP 模式） |
| 2 | 小程序连接设备 WiFi |
| 3 | 发送家庭 WiFi 配置（SmartConfig） |
| 4 | 设备连接家庭 WiFi |
| 5 | 设备广播自己的信息（UDP/mDNS） |

### Phase 4: 发现设备

| 方式 | 说明 |
|:---|:---|
| **局域网扫描** | UDP/mDNS 发现局域网内的设备 |
| **后端列表** | `GET /api/devices` 返回已绑定设备 |

### Phase 5: 绑定设备

| API | 参数 | 认证 |
|:---|:---|:---|
| `POST /api/devices/bind` | `{ macAddress, deviceName, plantId }` | Bearer Token |

**后端逻辑**：
1. `user_id` 从 token 获取
2. 通过 `macAddress` 查询设备
3. 设备不存在则自动创建
4. 更新 `Plant.current_device_id`

### Phase 6: 数据上报

| API | 参数 | 认证 |
|:---|:---|:---|
| `POST /api/devices/data` | `{ macAddress, metrics }` | 设备认证 |

**后端逻辑**：
1. 通过 `macAddress` 查询设备
2. 反向查询 `Plant.current_device_id` 获取 `plantId`
3. 创建环境读数记录
4. 更新设备心跳时间

## 关键设计点

| 设计点 | 说明 |
|:---|:---|
| **设备标识** | 使用 `macAddress`（设备固有），不使用 `deviceId`（后端生成） |
| **用户归属** | `user_id` 从 token 获取，不从前端参数传入 |
| **数据关联** | 单向关联，`Plant.current_device_id` → `Device` |
| **设备发现** | 前端 UDP/mDNS 扫描局域网（MVP 暂用后端接口） |
| **数据上报** | 设备使用 `macAddress`，后端反向查询获取 `plantId` |
