## 一、工作区概述

本工作区用于对 PlantGPT 项目进行**系统性运维监控**，通过日志分析、性能监控、异常检测等手段，保障系统稳定运行。

### 监控范围
- **前端**: 微信小程序客户端日志 (`client_logs`)
- **后端**: Node.js/Express 系统日志 (`system_logs`)
- **数据库**: MySQL 性能指标
- **业务**: API接口、定时任务、用户行为

## 四、数据源

### 后端日志表 (`system_logs`)
存储后端系统日志，包含以下关键字段：
- `level`: 日志级别 (debug/info/warn/error/fatal)
- `source`: 来源模块
- `request_id`: 请求追踪ID
- `url`/`method`: 请求信息
- `error_stack`: 错误堆栈
- `created_at`: 创建时间

### 前端日志表 (`client_logs`)
存储微信小程序客户端日志：
- `user_id`: 用户ID
- `page_path`: 页面路径
- `action`: 用户行为
- `device_info`: 设备信息
- `network_type`: 网络类型

## 六、快速开始

### 1. 执行单路径监控
```
1. 进入对应路径目录 (02-执行记录/路径X-xxx/)
2. 执行监控查询 (使用 MCP MySQL 或脚本)
3. 分析监控数据
4. 记录发现的问题
5. 输出执行记录文档
```

### 2. 常用查询
```sql
-- 查询最近错误
SELECT level, message, source, created_at 
FROM system_logs 
WHERE level IN ('error', 'fatal') 
ORDER BY created_at DESC LIMIT 50;

-- 按模块统计错误
SELECT source, COUNT(*) as count 
FROM system_logs 
WHERE level = 'error' 
  AND created_at >= DATE_SUB(NOW(), INTERVAL 1 HOUR)
GROUP BY source 
ORDER BY count DESC;
```
