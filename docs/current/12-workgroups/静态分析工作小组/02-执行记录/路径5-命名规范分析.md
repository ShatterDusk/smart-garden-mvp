## 命名转换架构

### ✅ 三层命名转换架构清晰

```
数据库层 (snake_case)
    ↓ Sequelize getterMethods
Model 层 (camelCase via getterMethods)
    ↓ Service 层手动映射或直接访问
Controller 层 (camelCase)
    ↓ API 响应
前端 (camelCase)
```

### 📝 映射模式说明

Service 层使用 `fieldMapping` 是因为 `update()` 方法需要传入 snake_case 字段名：

```javascript
// PlantService.js
const fieldMapping = {
  nickname: 'nickname',           // 无需转换
  plantCategory: 'plant_category', // 需要转换
  // ...
};

await plant.update({
  [fieldMapping.nickname]: updateData.nickname,
  [fieldMapping.plantCategory]: updateData.plantCategory,
});
```

return {

};
