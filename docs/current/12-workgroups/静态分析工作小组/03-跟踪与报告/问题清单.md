### 🔴 新增高危问题（性能相关）

#### SA-7-001: 会话功能响应时间过长
- **分析路径**: 路径7 - 性能与资源泄漏分析
- **文件位置**: 
  - `backend/server/src/controllers/sessionController.js:L267-421` (sendMessage)
  - `backend/server/src/services/aiService.js:L414-516` (analyze)
- **问题描述**: 
  - AI 服务采用同步阻塞调用，超时时间55秒
  - 用户发送消息后必须等待 AI 完整响应才能收到回复
  - 图片处理（下载+压缩）额外增加2-5秒耗时
- **风险影响**: 
  - 用户体验差，长时间等待无响应
  - 前端可能因超时而断开连接
  - 并发量大时连接池耗尽
- **根因分析**:
  1. **同步AI调用**: `aiService.analyze()` 同步等待AI响应
  2. **图片处理**: 每次请求都下载+压缩图片
  3. **上下文准备**: 每次查询数据库获取植物档案、环境数据
- **修复建议**: 
  - **P0**: 改为异步队列模式（Bull/Redis），立即返回"分析中"状态
  - **P1**: 图片上传时预压缩，避免实时处理
  - **P2**: 使用Redis缓存上下文数据（5分钟TTL）
  - **P3**: 实现WebSocket推送AI回复
- **参考文档**: `docs/current/11-knowledge/project-insights/solutions/session-performance-issue.md`
- **状态**: ⏳ 待修复

### 🟠 新增中危问题（定时任务相关）

#### SA-8-001: 定时任务无并发控制
- **分析路径**: 路径8 - 定时任务与异步流程分析
- **文件位置**: `backend/server/src/jobs/environmentSyncJob.js:L254-265`
- **问题描述**: 
  - `runSync()` 函数执行时间可能超过2小时（调度周期）
  - 无任务重叠保护，上一次未完成时下一次会启动
  - 可能导致数据库连接池耗尽、重复生成任务
- **风险影响**: 
  - 数据库连接池耗尽
  - 重复生成相同的 reading_tasks
  - 天气 API 重复调用
- **修复建议**: 
  ```javascript
  let isRunning = false;
  async function runSync() {
    if (isRunning) {
      logger.warn('上一次同步任务尚未完成，跳过本次执行');
      return;
    }
    isRunning = true;
    try {
      // ... 原有逻辑 ...
    } finally {
      isRunning = false;
    }
  }
  ```
- **状态**: ⏳ 待修复

#### SA-8-002: 无优雅关闭处理
- **分析路径**: 路径8 - 定时任务与异步流程分析
- **文件位置**: `backend/server/src/app.js`
- **问题描述**: 
  - 应用启动定时任务，但未注册 SIGTERM/SIGINT 处理
  - 容器化部署时，收到停止信号任务可能中断
- **风险影响**: 
  - 数据不一致（任务执行到一半被中断）
  - 数据库连接未正确关闭
- **修复建议**: 
  ```javascript
  process.on('SIGTERM', async () => {
    environmentSyncJob.stop();
    await sequelize.close();

  });
  ```
- **状态**: ⏳ 待修复
