## 元信息
- **文档类型**: 临时变更管理
- **版本**: V1.0
- **创建日期**: 2026-04-11
- **用途**: 追踪所有临时性代码变更，防止遗忘

---

## 一、什么是临时变更

临时变更是指：
- 调试时强制指定的环境配置
- 临时绕过的认证/校验
- 临时开启/关闭的功能开关
- 临时的日志级别调整
- 其他非永久性的代码修改

**风险**: 临时变更容易被遗忘，可能带入生产环境造成问题。

---

## 二、临时变更登记规范

### 2.1 登记时机
- 任何非修复问题的临时代码修改都必须登记
- 在修改代码前或修改后立即登记

### 2.2 登记内容

```markdown
### TC-{序号}: {变更简述}
- **变更类型**: 环境配置 / 认证绕过 / 功能开关 / 日志调整 / 其他
- **变更文件**: `文件路径`
- **变更内容**: 简述修改了什么
- **变更原因**: 为什么需要这个临时变更
- **创建时间**: YYYY-MM-DD HH:mm
- **创建人**: 
- **预计恢复时间**: YYYY-MM-DD HH:mm
- **状态**: 🟡 进行中 / ✅ 已恢复
- **恢复确认**: 
  - [ ] 代码已恢复
  - [ ] 功能验证通过
  - [ ] 已删除本记录
```

### 2.3 变更编号规则
- **TC**: Temporary Change（临时变更）
- **序号**: 从 001 开始递增
- 示例: `TC-001`, `TC-002`

## 五、临时变更检查清单

## 六、快速命令

```bash
# 查找代码中的临时变更标记
grep -r "【临时" --include="*.js" --include="*.ts" ../..

# 查找 TODO/FIXME/HACK 标记
grep -r "TODO\|FIXME\|HACK\|XXX" --include="*.js" --include="*.ts" ../..
```
