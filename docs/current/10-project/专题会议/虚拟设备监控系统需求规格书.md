# 虚拟设备监控系统需求规格书

**版本**: V1.0
**日期**: 2026-04-08
**状态**: 草案

---

## 1. 文档概述

### 1.1 目的

本文档定义虚拟设备监控系统的功能需求、性能指标和技术规范，为开发团队提供明确的实现指导。

### 1.2 范围

涵盖虚拟设备模拟器的Web监控界面、数据采集算法、场景模拟、设备通信等核心功能。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| **采集** | 虚拟设备生成传感器数据的过程 |
| **上报** | 将采集的数据发送到后端服务器 |
| **场景** | 预设的环境条件组合（温度、湿度等范围） |
| **SSE** | Server-Sent Events，服务器推送技术 |

## 2. 功能需求

### 2.1 设备状态监控 (REQ-001)

#### 2.1.1 需求描述

系统应实时显示虚拟设备的核心状态信息。

#### 2.1.2 功能点

| 功能点 | 优先级 | 验收标准 |
|--------|--------|----------|
| 显示设备ID | P0 | 显示完整的设备标识符 |
| 显示MAC地址 | P0 | 显示MAC地址，支持复制 |
| 显示设备名称 | P0 | 显示可读性设备名称 |
| 显示植物ID | P1 | 显示绑定的植物ID或"未绑定" |
| 显示WiFi状态 | P1 | 显示等待配网/已上线/配网失败 |
| 显示绑定状态 | P1 | 显示已绑定/未绑定 |
| 显示运行模式 | P2 | 显示手动模式/自动模式 |

#### 2.1.3 界面布局

```
┌─────────────────────────────────────┐
│ 设备状态                            │
├──────────────┬──────────────────────┤
│ 设备ID       │ MAC地址              │
│ [VIRTUAL_xx] │ [VIRTUAL_xx]         │
├──────────────┼──────────────────────┤
│ 设备名称     │ 植物ID               │
│ [proj-alpha] │ [未绑定]             │
├──────────────┼──────────────────────┤
│ WiFi状态     │ 绑定状态             │
│ [等待配网]   │ [未绑定]             │
├──────────────┴──────────────────────┤
│ 运行模式: 手动模式                   │
└─────────────────────────────────────┘
```

### 2.2 实时传感器数据 (REQ-002)

#### 2.2.1 需求描述

系统应实时显示8个传感器指标的数据，每秒更新。

#### 2.2.2 传感器指标

| 指标 | 代码 | 单位 | 正常范围 | 显示格式 |
|------|------|------|----------|----------|
| 温度 | temperature | °C | 20-30 | 1位小数 |
| 湿度 | humidity | % | 40-70 | 1位小数 |
| 气压 | pressure | hPa | 1000-1020 | 1位小数 |
| 光照 | light_intensity | lux | 1000-50000 | 整数 |
| 土壤湿度 | soil_moisture | % | 30-70 | 1位小数 |
| 土壤温度 | soil_temperature | °C | 18-28 | 1位小数 |
| 土壤pH | soil_ph | pH | 5.5-7.5 | 1位小数 |
| 电池电量 | battery_level | % | 0-100 | 整数 |

#### 2.2.3 数据更新要求

- **更新频率**: 每秒1次（通过SSE推送）
- **数据精度**: 按上表格式显示
- **异常标识**: 超出场景范围时红色高亮

### 2.3 场景模拟 (REQ-003)

#### 2.3.1 需求描述

提供8种预设场景，用户可点击切换，数据立即跳变到新场景范围。

#### 2.3.2 场景定义

```yaml
scenarios:
  normal:      # 正常环境
    temperature: [20, 30]
    humidity: [40, 70]
    pressure: [1000, 1020]
    light_intensity: [1000, 50000]
    soil_moisture: [30, 70]
    soil_temperature: [18, 28]
    soil_ph: [5.5, 7.5]
    
  drought:     # 干旱模式
    temperature: [30, 40]
    humidity: [10, 30]
    soil_moisture: [5, 20]
    
  high_temp:   # 高温模式
    temperature: [35, 45]
    humidity: [20, 40]
    
  low_temp:    # 低温模式
    temperature: [0, 10]
    humidity: [30, 50]
    
  high_hum:    # 高湿模式
    humidity: [80, 100]
    soil_moisture: [70, 95]
    
  low_light:   # 光照不足
    light_intensity: [0, 1000]
    
  watering:    # 浇水模式
    soil_moisture: [80, 100]
    humidity: [60, 90]
    
  night:       # 夜间模式
    light_intensity: [0, 50]
    temperature: [15, 25]
```

#### 2.3.3 切换行为

- **切换响应时间**: < 100ms
- **数据跳变**: 立即跳变到新场景随机值
- **UI反馈**: 按钮高亮 + 日志记录

### 2.4 设备配置 (REQ-004)

#### 2.4.1 需求描述

提供滑块控制采集间隔和上报间隔。

#### 2.4.2 配置项

| 配置项 | 范围 | 默认值 | 步长 |
|--------|------|--------|------|
| 采集间隔 | 1-30秒 | 5秒 | 1秒 |
| 上报间隔 | 5-300秒 | 60秒 | 5秒 |

#### 2.4.3 交互要求

- **实时预览**: 拖动时显示当前值
- **即时生效**: 释放后立即生效
- **边界检查**: 上报间隔 ≥ 采集间隔

### 2.5 实时日志 (REQ-005)

#### 2.5.1 需求描述

显示三类日志：UDP通信、后端通信、Web操作。

#### 2.5.2 日志类型

**UDP通信日志**
- 设备发现请求
- WiFi配置接收
- 心跳包

**后端通信日志**
- 数据上报请求/响应
- 设备绑定请求/响应
- 错误信息

**Web操作日志**
- 场景切换
- 配置变更
- 手动上报

#### 2.5.3 日志格式

```
[HH:MM:SS] [LEVEL] [SOURCE] Message
```

示例：
```
[14:30:25] [INFO] [UDP] 收到发现请求 from 192.168.1.100
[14:30:30] [INFO] [HTTP] POST /devices/data 200 OK
[14:30:35] [INFO] [WEB] 场景切换: 正常环境 → 高温模式
```

#### 2.5.4 显示要求

- **最大行数**: 100行
- **自动滚动**: 新日志自动滚动到底部
- **颜色区分**: INFO绿色、WARNING黄色、ERROR红色
