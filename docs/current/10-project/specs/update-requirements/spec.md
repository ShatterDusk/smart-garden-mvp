# 更新需求规格说明书 - 规格说明书

## 背景

经过审核发现，00-需求规格说明书中存在以下与实际前端实现不符的内容：

1. 植物列表筛选排序功能描述不完整
2. 诊断历史功能完全遗漏
3. 会话升级功能描述不完整
4. 设置页面功能未提及
5. 快速分析4步骤流程未明确
6. 环境数据源切换功能未明确

## 目标

补充和完善00-需求规格说明书，使其与实际前端功能完全一致。

## 需要补充的内容

### 1. 植物列表功能补充（PR-002）

**当前状态**：只提到"植物列表展示（卡片形式）"

**需要补充**：
- 分类筛选（多肉/花卉/绿植/全部）
- 搜索功能（按昵称、品种搜索）
- 排序功能（最近更新/健康度/添加时间）

### 2. 诊断历史功能（新增 PR-004）

**当前状态**：完全遗漏

**需要新增**：
- 诊断历史列表展示
- 诊断卡片详情（问题、建议、置信度）
- 诊断时间线
- 与消息关联的诊断记录

### 3. 会话升级功能补充（CS-001）

**当前状态**：术语已更新为"升级"，但流程描述不完整

**需要补充**：
- 升级触发条件（2个入口：快速分析后、咨询会话产生诊断卡后）
- 升级流程（创建植物档案 → 升级会话 → 保留历史）
- 升级后的行为变化

### 4. 设置页面功能（新增 UR-003）

**当前状态**：完全遗漏

**需要新增**：
- 通知设置（接收诊断提醒、养护提醒）
- 关于我们（版本信息、用户协议、隐私政策）
- 退出登录

### 5. 快速分析功能补充（AI-003）

**当前状态**：功能点列出，但流程不清晰

**需要补充**：
- 4步骤流程说明（选择植物→拍照→分析→结果）
- "继续对话"按钮功能
- 诊断结果展示内容

### 6. 环境数据功能补充（DV-002）

**当前状态**：功能点列出，但数据源切换未明确

**需要补充**：
- 数据源切换（设备数据/天气数据）
- 指标详情卡片展示
- 数据状态标识（适宜/偏高/偏低）
