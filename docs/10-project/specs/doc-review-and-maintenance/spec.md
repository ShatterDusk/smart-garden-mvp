## 一、背景与问题

### 1.1 问题描述

当前文档存在以下问题：

| 问题类型 | 示例 | 影响 |
|:---|:---|:---|
| 空占位文档 | 安全设计.md、安全规范.md、数据字典.md、变更记录.md | 无法指导开发 |
| 内容过时 | 部分文档版本号与代码不同步 | 误导开发 |
| 引用路径错误 | `02-architecture` 引用不存在的 `业务流程/README.md` | 文档无法导航 |
| 文档缺失 | 缺少独立的用户故事文档 | 需求追踪困难 |
| 内容重复 | 枚举值定义分散在多处 | 维护成本高 |

### 1.2 目标

- **查漏**：识别缺失的文档章节和内容
- **补缺**：补充占位文档或标记为"暂不需"
- **审核**：验证文档与代码的一致性

## 二、评审范围

### 2.1 文档分类

| 类别 | 目录 | 核心文档 |
|:---|:---|:---|
| 产品 | 01-product | 需求规格说明书 |
| 架构 | 02-architecture | 系统架构设计、数据库设计、API设计 |
| 前端 | 03-frontend | 前端页面设计 |
| 后端 | 04-backend | 服务架构、API参考 |
| 流程 | 05-process | 用户管理流程、AI交互流程 |

| 任务 | 08-tasks | backlog/active |
| 参考 | 09-references | 术语表、工作区模式 |
| 项目 | 10-project | 各类 spec 文档 |

### 2.2 评审维度

| 维度 | 说明 | 检查要点 |
|:---|:---|:---|
| **完整性** | 文档章节是否齐全 | 必有章节是否有内容、空占位是否处理 |
| **准确性** | 内容是否正确 | 技术方案是否可行、需求描述是否清晰 |
| **一致性** | 文档之间、文档与代码是否一致 | 命名规范、接口定义、数据结构 |
| **时效性** | 是否与最新实现同步 | 版本号、变更记录、实现状态 |
| **可用性** | 文档是否可操作 | 链接是否有效、路径是否正确 |

## 三、评审方案

### 3.1 三阶段评审

```
┌─────────────────────────────────────────────────────────────┐
│ 阶段一：快速扫描（2小时）                                         │
│ ├── 识别所有空占位文档（状态为"占位待完善"）                            │
│ ├── 识别版本过期文档（比代码版本旧）                                  │
│ ├── 识别引用路径错误                                              │
│ └── 输出：问题清单 v1.0                                           │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段二：深度审核（按模块）（8小时）                                     │
│ ├── 01-product: PRD 完整性检查                                    │
│ ├── 02-architecture: 架构与代码一致性                             │
│ ├── 03-frontend: 页面设计与实现对照                                │
│ └── 输出：问题清单 v2.0 + 修复建议                                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段三：修复与验收（按优先级）（4小时）                                  │
│ ├── P0: 空占位文档处理                                           │
│ ├── P1: 引用路径修复                                             │
│ └── P2: 内容更新（版本同步）                                        │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 优先级定义

| 优先级 | 定义 | 处理方式 |
|:---|:---|:---|
| P0 | 阻塞开发 | 立即处理或标记"暂不需" |
| P1 | 影响效率 | 本周内处理 |
| P2 | 长期优化 | 记录到债务清单，后续迭代处理 |

## 六、文档版本同步策略

### 6.1 同步规则

| 变更类型 | 文档版本更新 | 变更记录 |
|:---|:---|:---|
| 功能新增/修改 | 主版本号 +1 | 必须 |
| 内容修正/补充 | 次版本号 +1 | 必须 |
| 格式调整/错别字 | 修订号 +1 | 可选 |

### 6.2 版本号标注位置

每个文档顶部元数据：
```markdown
**版本**: V1.0
**日期**: 2026-04-16
**状态**: ✅ 已发布
```
