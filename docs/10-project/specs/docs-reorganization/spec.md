## 关键修正点

### 1. 字段命名统一

| 位置 | 当前问题 | 修正方案 |
|:---|:---|:---|
| 请求结构 | 使用 `cardId` | 改为 `messageId`（消息ID） |
| 返回结构 | `diagnosisCardId` 混淆 | 明确区分 `messageId` 和 `diagnosisCardId` |
| 数据库映射 | 缺少映射说明 | 增加 camelCase ↔ snake_case 对照表 |

### 2. 诊断卡生成规则（MVP简化）

```
简化前：多种条件判断（文字/图片/文字+图片）
简化后：唯一规则 - 有图就生成，没图不生成

confidence阈值：
- >= 0.5：正常显示诊断结果
- < 0.5：显示"无法识别"，提示重新拍照
```

### 3. 数据流向明确

```
用户发送消息 → 创建 message 记录 → 获得 message_id
    ↓
AI分析 → 创建 diagnosis_card 记录 → 获得 diagnosis_card_id
    ↓
返回：{ messageId, diagnosisCardId, diagnosis, ... }
```
