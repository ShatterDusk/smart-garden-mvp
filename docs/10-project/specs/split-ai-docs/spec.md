# 拆分AI交互设计文档 - 规格说明书

## 背景

当前 `04-AI交互与数据流转.md` 文档混合了两种内容：
1. **Mock实现**：描述前端实际的Mock数据实现流程
2. **应该的设计**：描述完整的后端架构设计

这种混合降低了文档的参考价值，需要拆分为两篇独立的文档。

## 目标

将现有文档拆分为两篇：
1. **04-AI交互与数据流转-实现.md**：描述现有的Mock实现（前端视角）
2. **04-AI交互与数据流转-设计.md**：描述应该的完整架构设计（后端视角）

## 拆分策略

### 文档1：实现篇（前端视角）

**目标读者**：前端开发人员

**内容结构**：
- 核心概念（会话类型、分析类型、上下文开关）
- 业务流程（Mock实现版本）
  - 咨询会话创建（2个入口）
  - 植物会话创建（2个入口）
  - 消息发送流程（Mock实现）
  - 数据流转架构（Mock实现）
- 前端状态管理
- 数据命名规范
- Mock数据接口说明

### 文档2：设计篇（后端视角）

**目标读者**：后端开发人员、架构师

**内容结构**：
- 核心概念（与实现篇一致）
- 系统架构
  - 服务拆分（API Gateway、会话服务、植物服务、环境服务、AI编排服务）
  - 数据层（MySQL、Redis、对象存储）
  - 外部服务（天气API、AI服务）
- 完整请求链路
  - 普通分析链路
  - 深度分析链路
- 服务职责划分
- API接口设计
- 数据库查询流程
- 演进路径
