# Redis 缓存改造方案

**方案版本**: v1.0  
**制定日期**: 2026-04-14  
**制定人**: AI Assistant

---

## 一、背景与目标

### 1.1 现状分析

当前系统使用内存缓存（Map）存储：
- 城市代码缓存（7天 TTL）
- 天气数据缓存（2小时 TTL）
- 天文数据缓存（24小时 TTL）

**存在的问题**：
1. 服务重启后缓存丢失
2. 多实例部署时缓存不共享
3. 无法水平扩展

### 1.2 改造目标

| 目标 | 说明 |
|:---|:---|
| 零侵入改造 | 不修改业务逻辑，只替换缓存实现 |
| 自动降级 | Redis 故障时自动回退到内存缓存 |
| 云托管兼容 | 支持微信云托管部署方案 |
| 渐进式切换 | 通过环境变量控制，随时可回滚 |

### 2.2 缓存抽象层设计

```javascript
// CacheService 职责：
// 1. 根据配置初始化对应的 Provider
// 2. 统一缓存操作接口（get/set/del/clear）
// 3. 错误处理和自动降级
// 4. 日志记录

interface CacheProvider {
  get(key: string): Promise<any>
  set(key: string, value: any, ttlMs: number): Promise<void>
  del(key: string): Promise<void>
  clear(): Promise<void>
}
```

## 四、部署方案

### 4.1 方案 A：微信云托管自建 Redis（推荐 MVP 阶段）

**适用场景**：缓存、Session、限流（可接受数据丢失）

**部署步骤**：

1. **创建 Redis 服务**
   ```bash
   # 使用 docker/Dockerfile 创建服务
   # 在微信云托管控制台 → 创建服务 → 从代码仓库部署
   ```

2. **配置环境变量**
   ```bash
   REDIS_HOST=redis-service  # 服务名
   REDIS_PORT=6379
   REDIS_PASSWORD=your-password
   ```

3. **服务间调用**
   ```javascript
   // 云托管内网调用
   const redis = new Redis({
     host: process.env.REDIS_HOST,  // 服务名作为主机名
     port: 6379,
     password: process.env.REDIS_PASSWORD,
   });
   ```

**成本**：低（仅需云托管资源费用）

### 4.2 方案 B：腾讯云 Redis + 资源互联

**适用场景**：生产环境、重要数据缓存

**部署步骤**：

1. **购买腾讯云 Redis**
   - 登录腾讯云控制台
   - 选择「上海」地域
   - 创建 Redis 实例

2. **配置资源互联**
   - 微信云托管控制台 → 设置 → 环境设置 → 增值服务
   - 购买「互联资源包」
   - 授权 VPC 绑定

3. **配置环境变量**
   ```bash
   REDIS_HOST=10.x.x.x  # 腾讯云 Redis 内网 IP
   REDIS_PORT=6379
   REDIS_PASSWORD=your-password
   ```

**成本**：中等（Redis 实例费用 + 互联资源包）

## 五、风险评估与应对

| 风险 | 影响 | 概率 | 应对措施 |
|:---|:---:|:---:|:---|
| Redis 连接失败 | 服务不可用 | 低 | 自动降级到内存缓存 |
| 缓存数据不一致 | 脏数据 | 中 | 合理设置 TTL，提供清除接口 |
| 改造引入 Bug | 功能异常 | 低 | 完整单元测试，渐进式上线 |
| 性能下降 | 响应变慢 | 低 | 本地缓存 + Redis 二级缓存 |
