## 基础信息

| 项目 | 值 |
|------|-----|
| 基础地址 | `http://localhost:3000/api/logs` |
| 访问密钥 | `dev-log-key-2024`（可通过环境变量 `LOG_ACCESS_KEY` 配置） |

## 通用响应格式

### 成功响应
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

### 错误响应
```json
{
  "code": 403,
  "message": "无效的访问密钥",
  "data": null
```

### 错误码说明

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 400 | 请求参数错误 |
| 403 | 访问密钥无效 |
| 404 | 日志文件不存在 |
| 500 | 服务器内部错误 |

## 接口列表

### 1. 获取日志文件列表

获取指定来源（后端/前端）的日志文件列表。

#### 请求

```
GET /api/logs/files?accessKey={accessKey}&source={source}
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| accessKey | string | 是 | 访问密钥 |
| source | string | 否 | 日志来源：`backend` 或 `frontend`，默认 `backend` |

#### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "files": [
      {
        "name": "all.log",
        "size": 1024567,
        "modifiedAt": "2025-01-09T10:30:00.000Z"
      },
      {
        "name": "error.log",
        "size": 45678,
        "modifiedAt": "2025-01-09T09:15:00.000Z"
      }
    ],
    "source": "backend",
    "path": "F:\\PROJECTS\\WeChatProjects\\MVP\\backend\\server\\logs"
  }
}
```

#### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| files | array | 日志文件列表 |
| files[].name | string | 文件名 |
| files[].size | number | 文件大小（字节） |
| files[].modifiedAt | string | 最后修改时间（ISO 8601） |
| source | string | 日志来源 |
| path | string | 日志目录绝对路径 |

---

### 2. 获取日志内容

获取指定日志文件的内容，支持返回最后 N 行。

#### 请求

```
GET /api/logs/content?accessKey={accessKey}&source={source}&file={file}&lines={lines}
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| accessKey | string | 是 | 访问密钥 |
| source | string | 否 | 日志来源：`backend` 或 `frontend`，默认 `backend` |
| file | string | 是 | 日志文件名（如 `all.log`） |
| lines | number | 否 | 返回最后 N 行，默认 100，最大 1000 |

#### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "file": "all.log",
    "source": "backend",
    "totalLines": 5000,
    "lines": 100,
    "content": "2025-01-09 10:00:00 info: 用户登录成功 {\"userId\":\"USER_xxx\"}\n2025-01-09 10:01:00 error: 数据库连接失败 {...}"
  }
}
```

#### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| file | string | 文件名 |
| source | string | 日志来源 |
| totalLines | number | 文件总行数 |
| lines | number | 实际返回行数 |
| content | string | 日志内容（换行符分隔） |

---

### 3. 搜索日志

在指定日志文件中搜索包含关键词的行。

#### 请求

```
GET /api/logs/search?accessKey={accessKey}&source={source}&file={file}&keyword={keyword}&lines={lines}
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| accessKey | string | 是 | 访问密钥 |
| source | string | 否 | 日志来源：`backend` 或 `frontend`，默认 `backend` |
| file | string | 是 | 日志文件名 |
| keyword | string | 是 | 搜索关键词 |
| lines | number | 否 | 返回匹配行数，默认 50 |

#### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "file": "all.log",
    "source": "backend",
    "keyword": "error",
    "matchedCount": 25,
    "content": "2025-01-09 10:01:00 error: 数据库连接失败 {...}\n2025-01-09 10:05:00 error: 请求超时 {...}"
  }
}
```

#### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| file | string | 文件名 |
| source | string | 日志来源 |
| keyword | string | 搜索关键词 |
| matchedCount | number | 匹配行数 |
| content | string | 匹配的日志内容 |

---

### 4. 清空日志文件

清空指定日志文件的内容（文件保留，内容清空）。

#### 请求

```
DELETE /api/logs/clear?accessKey={accessKey}&source={source}&file={file}
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| accessKey | string | 是 | 访问密钥 |
| source | string | 否 | 日志来源：`backend` 或 `frontend`，默认 `backend` |
| file | string | 是 | 日志文件名 |

#### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": "日志文件已清空",
    "file": "all.log",
    "source": "backend"
  }
}
```

## 前端调用示例

### 获取后端日志文件列表

```javascript
const logApi = require('./utils/logApi.js');

logApi.getBackendLogFiles()
  .then(data => {
    console.log('日志文件列表:', data.files);
  })
  .catch(err => {
    console.error('获取失败:', err.message);
  });
```

### 获取后端日志内容

```javascript
logApi.getBackendLogContent('all.log', 50)
  .then(data => {
    console.log('日志内容:', data.content);
  });
```

### 搜索后端日志

```javascript
logApi.searchBackendLogs('all.log', 'error', 20)
  .then(data => {
    console.log(`找到 ${data.matchedCount} 条匹配记录`);
    console.log(data.content);
  });
```

### 获取前端日志

```javascript
// 获取前端日志文件列表
logApi.getFrontendLogFiles()
  .then(data => {
    console.log('前端日志文件:', data.files);
  });

// 获取前端日志内容
logApi.getFrontendLogContent('frontend-2025-01-09.log', 100)
  .then(data => {
    console.log('前端日志:', data.content);
  });
```

## 注意事项

1. **密钥安全**：生产环境请修改默认密钥 `dev-log-key-2024`
2. **路径安全**：接口已做目录遍历防护，无法访问日志目录外的文件
3. **前端日志路径**：前端日志位于 `frontend/logs/` 目录，需确保该目录存在
4. **性能考虑**：大日志文件搜索可能较慢，建议限制返回行数
