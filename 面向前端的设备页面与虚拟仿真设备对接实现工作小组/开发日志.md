#### ✅ VD-003: 修复虚拟设备数据上报认证问题
**状态**: 已完成  
**工时**: 30分钟

**问题描述**:
虚拟设备在手动模式下（前端绑定）无法上报数据，因为 `deviceAuthMiddleware` 会检查数据库中是否存在该 deviceId，而手动模式下虚拟设备还没有 device_id。

**解决方案**:
在 `deviceAuthMiddleware` 中添加开发环境特殊处理：
- 当 `NODE_ENV=development` 且 deviceId 以 `VIRTUAL_` 开头时，直接通过认证
- 这样虚拟设备在开发环境可以正常上报数据，不影响生产环境的安全性

**代码变更**:
```javascript
// backend/server/src/middleware/deviceAuth.js
// 开发环境：允许虚拟设备通过认证
if (process.env.NODE_ENV === 'development' && deviceId.startsWith('VIRTUAL_')) {
  logger.info('开发环境：虚拟设备认证通过', { deviceId });
  req.device = {
    deviceId: deviceId,
    status: 'online',
    boundPlantId: req.body.plantId || null,
  };
  return next();
}
```

```python
def _init_manual_mode(self):
    """初始化手动模式（前端绑定模式）"""
    if not self.mac_address:
        self.mac_address = f"VIRTUAL_{random.randint(10000000, 99999999)}"
    if not self.device_name:
        self.device_name = f"proj-alpha-虚拟设备-{self.mac_address[-4:]}"
    self.device_id = None
    self.plant_id = None
    self.token = None

```

```
4. 选择设备并配置WiFi

5. 虚拟设备处理配置

   - 启动数据上报

6. 前端检测设备上线
   - 定期发送 discover 广播
   - 检测到设备状态为 online
   - 显示配网成功

7. 绑定设备
```
