# 03-技术选型

## 1. 编程语言

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| Python | 3.9+ | 生态丰富、开发效率高、asyncio支持好 |

## 2. 核心框架

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| asyncio | 异步编程 | 标准库、性能优秀、生态成熟 |
| aiohttp | HTTP客户端/服务器 | 异步支持、性能高 |
| Flask | Web框架 | 轻量、易用、SSE支持好 |
| Pydantic | 数据验证 | 类型安全、自动生成文档 |

## 3. 数据层

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| NumPy | 数值计算 | 性能高、科学计算标准 |
| Pandas | 数据处理 | 数据分析、时间序列 |

## 4. 基础设施

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| structlog | 结构化日志 | 结构化、可过滤、性能好 |
| PyYAML | 配置解析 | 标准格式、易读易写 |
| python-dotenv | 环境变量 | 开发友好、配置分离 |

## 5. 测试

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| pytest | 测试框架 | 功能强大、插件丰富 |
| pytest-asyncio | 异步测试 | asyncio支持 |
| pytest-cov | 覆盖率 | 集成简单 |
| httpx | HTTP测试 | 异步支持、API友好 |

## 6. 前端（Web界面）

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| Vanilla JS | 交互逻辑 | 轻量、无依赖 |
| Chart.js | 数据可视化 | 易用、美观 |
| Tailwind CSS | 样式 | 实用优先、响应式 |

## 7. 备选方案

| 场景 | 方案A（当前） | 方案B（备选） | 选择理由 |
|------|--------------|--------------|----------|
| Web框架 | Flask | FastAPI | Flask更简单，FastAPI性能更好 |
| 数据库 | SQLite | PostgreSQL | SQLite零配置，适合单机 |
| 缓存 | 内存 | Redis | 内存简单，Redis支持分布式 |

## 8. 版本约束

```
Python >= 3.9
Flask >= 2.0
aiohttp >= 3.8
Pydantic >= 1.10
structlog >= 22.0
```
