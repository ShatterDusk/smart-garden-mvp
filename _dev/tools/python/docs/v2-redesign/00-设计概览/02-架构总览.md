# 02-架构总览

## 1. 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Virtual Device V2                         │
├─────────────────────────────────────────────────────────────┤
│  Presentation Layer (表现层)                                  │
│  ├── Web Interface (Flask + SSE)                             │
│  └── CLI Interface (可选)                                     │
├─────────────────────────────────────────────────────────────┤
│  Application Layer (应用层)                                   │
│  ├── Device Manager (设备管理器)                              │
│  ├── Event Timeline (事件时间线)                              │
│  └── API Gateway (API网关)                                    │
├─────────────────────────────────────────────────────────────┤
│  Domain Layer (领域层)                                        │
│  ├── Device Instance (设备实例)                               │
│  ├── Data Generator (数据生成器)                              │
│  └── State Machine (状态机)                                   │
├─────────────────────────────────────────────────────────────┤
│  Infrastructure Layer (基础设施层)                            │
│  ├── UDP Service (UDP服务)                                    │
│  ├── HTTP Client (HTTP客户端)                                 │
│  ├── Config Manager (配置管理)                                │
│  ├── Log Manager (日志管理)                                   │
│  └── State Store (状态存储)                                   │
└─────────────────────────────────────────────────────────────┘
```

## 2. 核心组件

### 2.1 Device Instance
- 代表一个虚拟设备
- 维护设备状态
- 管理数据生成
- 处理网络通信

### 2.2 Device Manager
- 管理多个设备实例
- 设备生命周期管理
- 设备发现服务

### 2.3 Data Generator
- 三层数据生成策略
- 插件化架构
- 自然模拟算法

### 2.4 Event Timeline
- 事件队列管理
- 虚拟时间系统
- 时间旅行调试

## 3. 数据流

```
[Device Instance] → [Data Generator] → [Sensor Data]
       ↓
[Event Timeline] → [Event Trigger] → [State Change]
       ↓
[UDP Service] ←→ [Frontend Discovery]
       ↓
[HTTP Client] → [Backend API]
```
