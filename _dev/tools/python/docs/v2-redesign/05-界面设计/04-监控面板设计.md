# 监控面板设计

## 1. 概述

监控面板是虚拟设备 V2 的核心界面，提供设备状态、传感器数据、场景信息和系统日志的实时监控功能。

## 7. 实时更新机制

### 7.1 WebSocket 数据流

```
WebSocket 连接
    ↓
接收数据推送
    ↓
解析数据类型
    ↓
├─> 传感器数据 → 更新对应卡片
├─> 场景状态 → 更新场景栏
├─> 时间线事件 → 更新时间线
└─> 系统日志 → 追加到日志列表
```

### 7.2 更新节流

```javascript
// 数据更新节流
class UpdateThrottler {
  constructor(minInterval = 100) {
    this.minInterval = minInterval;
    this.lastUpdate = {};
  }

  shouldUpdate(key) {
    const now = Date.now();
    if (!this.lastUpdate[key] || 
        now - this.lastUpdate[key] >= this.minInterval) {
      this.lastUpdate[key] = now;
      return true;
    }
    return false;
  }
}
```

### 8.2 数据缓存

```javascript
// 传感器数据缓存
const dataCache = {
  maxSize: 1000,
  data: new Map(),
  
  set(key, value) {
    if (this.data.size >= this.maxSize) {
      const firstKey = this.data.keys().next().value;
      this.data.delete(firstKey);
    }
    this.data.set(key, value);
  },
  
  get(key) {
    return this.data.get(key);
  }
};
```

## 9. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 更新策略 | WebSocket推送 | 实时性高 |
| 数据展示 | 卡片式 | 信息清晰 |
| 时间线 | 水平布局 | 符合阅读习惯 |
| 日志展示 | 列表+筛选 | 便于查找 |
| 性能优化 | 虚拟滚动 | 大数据量流畅 |
