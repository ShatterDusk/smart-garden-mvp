# 日志系统

## 1. 概述

日志系统提供结构化、可配置的日志记录功能，支持多种输出格式和级别，便于问题排查和系统监控。

## 2. 日志架构

```
┌─────────────────────────────────────────────────────────────┐
│                         日志系统                              │
├─────────────────────────────────────────────────────────────┤
│  应用代码  →  Logger  →  Handler  →  Formatter  →  输出目标   │
│                ↓           ↓           ↓                     │
│           日志级别      输出方式      格式定义                 │
│           (DEBUG)    (文件/控制台)   (JSON/Text)             │
└─────────────────────────────────────────────────────────────┘
```

## 3. 日志级别

| 级别 | 数值 | 使用场景 |
|------|------|----------|
| DEBUG | 10 | 详细调试信息 |
| INFO | 20 | 一般信息 |
| WARNING | 30 | 警告信息 |
| ERROR | 40 | 错误信息 |
| CRITICAL | 50 | 严重错误 |

## 4. 结构化日志

### 4.1 日志格式

```json
{
  "timestamp": "2026-04-08T10:30:00.123Z",
  "level": "INFO",
  "logger": "DeviceManager",
  "message": "Device started",
  "context": {
    "device_id": "vd_001",
    "device_name": "测试设备"
  },
  "source": {
    "file": "device_manager.py",
    "line": 45,
    "function": "start_device"
  },
  "trace_id": "trace_abc123"
}
```

### 4.2 日志配置

```yaml
logging:
  version: 1
  disable_existing_loggers: false
  
  formatters:
    json:
      class: pythonjsonlogger.jsonlogger.JsonFormatter
      format: '%(timestamp)s %(level)s %(name)s %(message)s'
    
    text:
      format: '%(asctime)s [%(levelname)s] %(name)s: %(message)s'
      datefmt: '%Y-%m-%d %H:%M:%S'
  
  handlers:
    console:
      class: logging.StreamHandler
      level: DEBUG
      formatter: text
      stream: ext://sys.stdout
    
    file:
      class: logging.handlers.RotatingFileHandler
      level: INFO
      formatter: json
      filename: logs/virtual-device.log
      maxBytes: 104857600  # 100MB
      backupCount: 5
      encoding: utf8
  
  loggers:
    virtual_device:
      level: DEBUG
      handlers: [console, file]
      propagate: false
    
    uvicorn:
      level: INFO
      handlers: [console]
      propagate: false
  
  root:
    level: WARNING
    handlers: [console]
```

## 5. 日志使用

### 5.1 基本用法

```python
import structlog

# 获取logger
logger = structlog.get_logger("virtual_device")

# 记录日志
logger.debug("Debug message", key="value")
logger.info("Device started", device_id="vd_001")
logger.warning("High temperature", temperature=45.2)
logger.error("Connection failed", error=str(e))
logger.critical("System crash", reason="out_of_memory")
```

### 5.2 上下文绑定

```python
# 绑定上下文
bound_logger = logger.bind(device_id="vd_001", user="admin")

# 后续日志自动包含上下文
bound_logger.info("Starting device")
# 输出: {"message": "Starting device", "device_id": "vd_001", "user": "admin"}

bound_logger.info("Device ready")
# 输出: {"message": "Device ready", "device_id": "vd_001", "user": "admin"}

# 清除绑定
new_logger = bound_logger.unbind("user")
```

## 6. 日志存储

### 6.1 文件轮转

```python
from logging.handlers import RotatingFileHandler, TimedRotatingFileHandler

# 按大小轮转
size_handler = RotatingFileHandler(
    "app.log",
    maxBytes=100*1024*1024,  # 100MB
    backupCount=5
)

# 按时间轮转
time_handler = TimedRotatingFileHandler(
    "app.log",
    when="midnight",
    interval=1,
    backupCount=30
)
```

### 6.2 日志清理

```python
import os
import glob
from datetime import datetime, timedelta

def cleanup_old_logs(log_dir: str, max_age_days: int = 30):
    """清理旧日志文件"""
    cutoff = datetime.now() - timedelta(days=max_age_days)
    
    for log_file in glob.glob(os.path.join(log_dir, "*.log*")):
        mtime = datetime.fromtimestamp(os.path.getmtime(log_file))
        if mtime < cutoff:
            os.remove(log_file)
            print(f"Removed old log: {log_file}")
```
