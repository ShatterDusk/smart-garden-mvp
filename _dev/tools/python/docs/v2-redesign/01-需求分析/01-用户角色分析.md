# 用户角色分析

## 概述

虚拟设备V2系统服务于多种用户角色，每种角色有不同的使用场景、技术能力和需求重点。

---

## 角色定义

### 1. 前端开发工程师 (Frontend Developer)

**特征描述**
- 主要工作：开发微信小程序设备相关页面
- 技术能力：熟悉微信小程序开发、JavaScript
- 对硬件了解有限，需要稳定的虚拟设备进行联调

**核心需求**
| 需求项 | 优先级 | 说明 |
|:---|:---|:---|
| 快速启动虚拟设备 | P0 | 一键启动，无需复杂配置 |
| 模拟真实设备流程 | P0 | 完整的发现→绑定→数据上报流程 |
| 可视化监控面板 | P1 | 实时查看设备状态和数据 |
| 调试信息输出 | P1 | 清晰的日志便于问题定位 |
| 多设备模拟 | P2 | 同时模拟多个设备测试列表页 |

**使用场景**
1. 开发设备列表页时，需要多个虚拟设备展示不同状态
2. 开发设备详情页时，需要虚拟设备持续上报数据
3. 调试WiFi配网流程时，需要模拟设备发现过程
4. 测试异常处理时，需要模拟设备离线、数据异常等情况

**痛点**
- 真实设备数量有限，无法覆盖所有测试场景
- 真实设备数据不可控，难以测试边界情况
- 真实设备配网流程繁琐，影响开发效率

---

### 2. 后端开发工程师 (Backend Developer)

**特征描述**
- 主要工作：开发设备管理后端API、数据处理逻辑
- 技术能力：熟悉Node.js、数据库、网络协议
- 需要验证后端逻辑对设备数据的处理是否正确

**核心需求**
| 需求项 | 优先级 | 说明 |
|:---|:---|:---|
| 协议一致性 | P0 | 严格遵循设备通信协议 |
| 数据格式验证 | P0 | 确保上报数据格式符合预期 |
| 异常场景模拟 | P1 | 模拟超时、重连、数据异常等 |
| 性能测试支持 | P2 | 模拟高并发设备连接 |
| 自动化测试集成 | P2 | 支持CI/CD流水线集成 |

**使用场景**
1. 开发设备绑定API时，验证请求参数处理逻辑
2. 开发数据存储逻辑时，验证数据解析和入库
3. 开发告警逻辑时，验证阈值判断和通知触发
4. 压力测试时，模拟大量设备并发上报

**痛点**
- 难以构造特定的测试数据验证边界条件
- 缺乏可重复的稳定测试环境
- 真实设备测试成本高昂

---

### 3. 测试工程师 (QA Engineer)

**特征描述**
- 主要工作：执行测试用例、编写自动化测试
- 技术能力：熟悉测试方法论、自动化测试工具
- 需要可重复、可预测的测试环境

**核心需求**
| 需求项 | 优先级 | 说明 |
|:---|:---|:---|
| 测试场景预设 | P0 | 快速切换到标准测试场景 |
| 事件编排能力 | P1 | 按时间线编排设备行为 |
| 断言支持 | P1 | 验证预期结果是否达成 |
| 测试报告生成 | P2 | 输出结构化的测试结果 |
| 回归测试套件 | P2 | 预置常用回归测试场景 |

**使用场景**
1. 执行功能测试时，使用预设场景快速验证
2. 执行集成测试时，编排多设备交互场景
3. 执行异常测试时，模拟设备故障和网络异常
4. 执行回归测试时，批量运行标准测试套件

**痛点**
- 手动构造测试数据耗时且易出错
- 复杂场景难以手动复现
- 缺乏自动化的结果验证机制

---

### 4. 产品经理/设计师 (Product Manager/Designer)

**特征描述**
- 主要工作：产品规划、交互设计
- 技术能力：非技术背景或轻度技术背景
- 关注用户体验和产品功能演示

**核心需求**
| 需求项 | 优先级 | 说明 |
|:---|:---|:---|
| 直观的数据展示 | P0 | 无需技术背景即可理解 |
| 场景演示模式 | P1 | 一键演示产品功能 |
| 数据导出分享 | P2 | 导出数据用于汇报 |
| 美观的界面 | P2 | 专业的产品演示效果 |

**使用场景**
1. 向客户演示产品时，展示设备数据效果
2. 评审设计方案时，验证交互流程
3. 撰写PRD时，参考真实数据格式

**痛点**
- 无法独立操作真实设备
- 缺乏直观的演示环境

---

### 5. DevOps/运维工程师 (DevOps Engineer)

**特征描述**
- 主要工作：系统部署、监控、运维
- 技术能力：熟悉Docker、K8s、CI/CD
- 关注系统的可部署性和可观测性

**核心需求**
| 需求项 | 优先级 | 说明 |
|:---|:---|:---|
| 容器化部署 | P0 | Docker镜像一键部署 |
| 配置外部化 | P1 | 通过环境变量配置 |
| 健康检查 | P1 | 提供健康检查端点 |
| 监控指标 | P2 | 暴露Prometheus指标 |
| 日志聚合 | P2 | 结构化日志便于收集 |

**使用场景**
1. 在CI/CD流水线中部署虚拟设备用于集成测试
2. 在生产环境部署用于演示或降级方案
3. 监控系统运行状态和性能指标

---

## 角色优先级矩阵

```
                    影响力
              低 ◄─────────────────► 高
         ┌─────────────────────────────┐
     高  │  产品经理      前端开发      │
         │  (P2)         (P0)          │
  紧     │                             │
  迫     │  测试工程师    后端开发      │
  性    │  (P1)         (P0)          │
         │                             │
     低  │  DevOps                     │
         │  (P2)                       │
         └─────────────────────────────┘
```

**优先级说明**：
- **P0**: 核心用户，必须完全满足需求
- **P1**: 重要用户，主要需求必须满足
- **P2**: 一般用户，在资源允许时满足

## 角色交互场景

### 场景1: 前端联调
```
前端开发 ──► 启动虚拟设备 ──► 查看监控面板 ──► 调试小程序页面
                │
                ▼
         后端开发(验证API)
```

### 场景2: 集成测试
```
测试工程师 ──► 加载测试场景 ──► 执行自动化测试 ──► 生成测试报告
                   │
                   ▼
            虚拟设备(按编排执行)
```

### 场景3: 产品演示
```
产品经理 ──► 选择演示场景 ──► 一键启动 ──► 向客户展示
```
