# 边界情况分析

## 概述

边界情况分析识别系统在各种极端和异常条件下的行为，确保系统健壮性。

---

## 1. 设备模拟边界

### 1.1 设备数量边界

| 场景 | 边界值 | 预期行为 | 处理方式 |
|:---|:---:|:---|:---|
| 创建第1个设备 | 1 | 正常创建 | 初始化设备管理器 |
| 创建第50个设备 | 50 | 正常创建 | 达到推荐上限，记录警告 |
| 尝试创建第51个设备 | 51 | 拒绝创建 | 返回错误：DEV_005 设备数量达到上限 |
| 创建第100个设备 | 100 | 拒绝创建（硬限制） | 返回错误，提示清理旧设备 |
| 所有设备同时停止 | 50→0 | 正常停止 | 批量操作，异步处理 |
| 所有设备同时启动 | 0→50 | 正常启动 | 分批启动，避免资源抢占 |

**资源限制策略**:
```python
# 软限制：警告但允许
SOFT_LIMIT = 50
# 硬限制：拒绝创建
HARD_LIMIT = 100

if device_count >= HARD_LIMIT:
    raise DeviceLimitExceededError("已达到最大设备数量限制")
elif device_count >= SOFT_LIMIT:
    logger.warning(f"设备数量({device_count})超过推荐值({SOFT_LIMIT})")
```

### 1.2 设备生命周期边界

| 场景 | 触发条件 | 预期行为 | 异常处理 |
|:---|:---|:---|:---|
| 快速创建-销毁循环 | 1秒内创建并销毁10次 | 资源正确释放 | 防止资源泄漏，延迟销毁 |
| 销毁运行中设备 | 设备正在上报数据时销毁 | 优雅停止，等待上报完成 | 5秒超时后强制停止 |

| 启动已销毁设备 | 设备实例已释放 | 返回设备不存在错误 | 检查设备存在性 |

### 1.3 网络边界

| 场景 | 边界条件 | 预期行为 | 恢复策略 |
|:---|:---|:---|:---|
| UDP端口被占用 | 8080端口已被其他程序使用 | 尝试8081-8090范围 | 记录实际使用端口 |
| HTTP端口被占用 | 5000端口被占用 | 尝试5001-5010范围 | 在启动日志中显示 |
| 后端服务器不可达 | 网络断开/IP错误 | 标记设备离线，缓存数据 | 指数退避重连 |
| 后端响应超时 | 30秒内无响应 | 标记请求失败，重试 | 最多重试3次 |
| 后端返回500错误 | 服务器内部错误 | 暂停上报，等待恢复 | 延长重试间隔 |
| WiFi配网超时 | 60秒内未完成配网 | 返回AP模式 | 记录失败原因 |
| 频繁切换WiFi | 1分钟内切换5次 | 限制切换频率 | 30秒冷却期 |

**重连策略**:
```python
# 指数退避算法
base_delay = 1  # 秒
max_delay = 60  # 秒
retry_count = 0

delay = min(base_delay * (2 ** retry_count), max_delay)
retry_count += 1
```

## 2. 数据生成边界

### 2.1 传感器数据边界

| 传感器 | 正常范围 | 边界值 | 超限处理 |
|:---|:---|:---|:---|
| 温度 | -40°C ~ 80°C | -50°C, 90°C | 标记为异常，记录警告 |
| 湿度 | 0% ~ 100% | -5%, 105% | 截断到有效范围 |
| 光照 | 0 ~ 100000 lux | -100, 150000 | 截断到有效范围 |
| 土壤湿度 | 0% ~ 100% | -10%, 110% | 截断到有效范围 |
| 电池 | 0% ~ 100% | -5%, 105% | 截断到有效范围 |

**数据验证规则**:
```python
def validate_sensor_data(data: dict) -> tuple[bool, list]:
    errors = []
    
    if 'temperature' in data:
        if not -40 <= data['temperature'] <= 80:
            errors.append(f"温度超出范围: {data['temperature']}")
    
    if 'humidity' in data:
        data['humidity'] = max(0, min(100, data['humidity']))
    
    return len(errors) == 0, errors
```

### 2.2 场景切换边界

| 场景 | 边界条件 | 预期行为 |
|:---|:---|:---|
| 快速场景切换 | 1秒内切换5次场景 | 只执行最后一次切换，忽略中间请求 |
| 切换到相同场景 | 当前已是目标场景 | 忽略操作，记录日志 |
| 场景切换中断 | 切换过程中停止设备 | 终止切换，保持当前状态 |
| 极端参数场景 | 温度范围超过传感器极限 | 截断到传感器有效范围 |
| 多设备同时切换 | 50设备同时切换场景 | 异步处理，避免阻塞 |

**场景切换防抖**:
```python
last_switch_time = 0
debounce_interval = 2  # 秒

def switch_scenario(device, new_scenario):
    global last_switch_time
    current_time = time.time()
    
    if current_time - last_switch_time < debounce_interval:
        logger.info("场景切换过于频繁，忽略本次请求")
        return
    
    last_switch_time = current_time
    # 执行场景切换
```

## 3. 事件时间线边界

### 3.1 事件调度边界

| 场景 | 边界条件 | 预期行为 | 处理方式 |
|:---|:---|:---|:---|
| 事件时间冲突 | 同一时间多个事件 | 按优先级执行 | 定义事件优先级 |
| 事件时间过期 | 事件时间已过 | 根据策略处理 | 跳过/立即执行/警告 |
| 高频事件 | 每秒10个事件 | 正常处理 | 使用异步队列 |
| 事件循环嵌套 | A触发B，B触发A | 检测并中断 | 最大嵌套深度10 |
| 事件执行超时 | 事件执行超过5分钟 | 强制中断 | 记录超时错误 |
| 事件参数过大 | 参数>1MB | 拒绝创建 | 返回参数过大错误 |

**事件优先级**:
| 优先级 | 事件类型 | 说明 |
|:---:|:---|:---|
| 1 (最高) | device_restart, sensor_fault | 系统级事件 |
| 2 | scenario_change | 场景切换 |
| 3 | watering, fertilizing | 用户操作 |
| 4 | temperature_spike | 环境变化 |
| 5 (最低) | custom | 自定义事件 |

## 7. 安全边界

### 7.1 输入验证边界

| 输入类型 | 边界检查 | 处理方式 |
|:---|:---|:---|
| 设备ID | 长度10-32，字母数字 | 正则验证，失败拒绝 |
| 设备名称 | 长度2-50，无特殊字符 | 转义处理，XSS防护 |
| JSON参数 | 深度<10，大小<1MB | 解析前检查 |
| 文件上传 | 类型白名单，<10MB | 沙箱处理 |
| URL参数 | 长度<2048 | 截断或拒绝 |

### 7.2 资源防护边界

| 资源 | 防护策略 | 触发行为 |
|:---|:---|:---|
| CPU | 使用率<90% | 降低时间加速倍数 |

| 磁盘 | 可用>100MB | 停止日志写入 |
| 网络 | 带宽<80% | 压缩数据，降低频率 |
| 句柄 | 文件句柄<1000 | 关闭空闲连接 |
