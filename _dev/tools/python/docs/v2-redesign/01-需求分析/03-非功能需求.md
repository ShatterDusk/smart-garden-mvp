# 非功能需求

## 概述

非功能需求定义系统的质量属性和约束条件，是衡量系统成功的重要标准。

---

## 1. 性能需求 (PERF)

### PERF-001: 响应时间

| 指标 | 目标值 | 可接受值 | 测量方法 |
|:---|:---|:---|:---|
| Web界面加载时间 | < 2秒 | < 5秒 | 从请求到首屏渲染 |
| API响应时间 | < 100ms | < 500ms | 从请求到响应完成 |
| 数据刷新延迟 | < 1秒 | < 3秒 | 数据生成到界面更新 |
| 场景切换时间 | < 500ms | < 2秒 | 点击到数据变化 |
| 日志刷新频率 | 实时 | < 1秒 | 日志产生到显示 |

**验收标准**:
- 95%的请求响应时间满足目标值
- 99%的请求响应时间满足可接受值
- 不存在超过5秒的请求（除文件上传/下载）

---

### PERF-002: 并发能力

| 指标 | 目标值 | 说明 |
|:---|:---|:---|
| 最大虚拟设备数 | 50个 | 单实例支持的设备数量 |
| 并发Web连接 | 100个 | SSE/WebSocket并发连接 |
| 数据上报QPS | 1000条/秒 | 所有设备合计 |
| 内存占用 | < 500MB | 50设备正常运行时 |
| CPU占用 | < 50% | 50设备正常运行时 |

**验收标准**:
- 50个设备同时运行，系统稳定不崩溃
- 内存使用增长线性，无内存泄漏
- CPU使用在合理范围内，不影响其他应用

---

### PERF-003: 时间加速性能

| 时间缩放倍数 | 最大设备数 | 说明 |
|:---|:---|:---|
| 1x (实时) | 50 | 正常速度运行 |
| 10x | 50 | 10倍速运行 |
| 60x | 30 | 1小时=1分钟 |
| 600x | 10 | 10小时=1分钟 |

**验收标准**:
- 时间加速时数据生成准确，无跳变
- 高倍速下界面仍保持响应
- 可平滑切换时间缩放倍数

---

## 2. 可靠性需求 (RELI)

### RELI-001: 系统可用性

| 指标 | 目标值 | 说明 |
|:---|:---|:---|
| 系统可用性 | 99.9% | 计划内停机除外 |
| 平均故障间隔 | > 720小时 | 30天无故障运行 |
| 平均恢复时间 | < 5分钟 | 从故障到恢复 |
| 数据丢失率 | 0% | 配置和脚本数据 |

**验收标准**:
- 连续运行30天无崩溃
- 异常情况下优雅降级，不丢失数据
- 自动恢复机制，无需人工干预

---

### RELI-002: 错误处理

| 场景 | 处理要求 |
|:---|:---|
| 后端连接失败 | 自动重连，指数退避，最大间隔60秒 |
| 网络异常 | 本地缓存数据，恢复后补报 |
| 配置错误 | 启动时校验，给出明确错误信息 |
| 资源不足 | 拒绝新设备创建，保持现有设备运行 |
| 未知异常 | 记录详细日志，不中断其他设备 |

**验收标准**:
- 任何单点故障不影响其他设备运行
- 错误信息清晰，包含上下文
- 提供故障排查指南

---

### RELI-003: 数据一致性

| 场景 | 一致性要求 |
|:---|:---|
| 时间线执行 | 事件按时间顺序执行，不丢失 |
| 场景切换 | 数据平滑过渡，无跳变 |
| 并发操作 | 设备状态变更原子性 |
| 持久化数据 | 配置、脚本、日志不丢失 |

**验收标准**:
- 事件执行顺序与定义一致
- 系统崩溃后，已保存数据可恢复
- 多设备并发操作无竞态条件

## 3. 可维护性需求 (MAINT)

### MAINT-001: 代码质量

| 指标 | 目标值 | 检查工具 |
|:---|:---|:---|
| 代码覆盖率 | > 80% | pytest-cov |
| 圈复杂度 | < 10 | radon |
| 代码重复率 | < 5% | pylint |
| 类型注解覆盖率 | > 90% | mypy |
| 文档覆盖率 | 100% | 公共API |

**验收标准**:
- 所有公共方法有docstring
- 核心逻辑有单元测试
- 代码通过lint检查

---

### MAINT-002: 可扩展性

| 扩展点 | 扩展方式 | 说明 |
|:---|:---|:---|
| 新传感器类型 | 插件接口 | 实现Sensor接口即可 |
| 新场景模式 | 配置文件 | 添加场景定义YAML |
| 新事件类型 | 继承基类 | 继承Event基类 |
| 新数据生成器 | 插件系统 | 注册到生成器工厂 |
| 新协议支持 | 实现接口 | 实现Protocol接口 |

**验收标准**:
- 新增传感器类型无需修改核心代码
- 插件加载失败不影响系统运行
- 提供插件开发文档和示例

---

### MAINT-003: 可配置性

| 配置项 | 配置方式 | 热加载 |
|:---|:---|:---:|
| 后端服务器地址 | 环境变量/配置文件 | ✓ |
| 日志级别 | 配置文件/API | ✓ |
| 设备默认参数 | 配置文件 | ✗ |
| 场景参数 | 配置文件/API | ✓ |
| 时间缩放 | API/界面 | ✓ |
| 网络端口 | 环境变量 | ✗ |

**验收标准**:
- 所有配置有默认值
- 配置变更记录日志
- 配置错误有明确提示

## 5. 安全性需求 (SEC)

### SEC-001: 通信安全

| 需求 | 实现方式 |
|:---|:---|
| HTTPS支持 | 后端API使用HTTPS |
| Token安全 | 设备Token不存储在日志 |
| 密码保护 | WiFi密码不显示在界面 |
| 会话管理 | Web界面会话超时30分钟 |

**验收标准**:
- 敏感信息不记录在日志
- 密码输入框使用掩码
- 会话过期自动登出

---

### SEC-002: 访问控制

| 需求 | 实现方式 |
|:---|:---|
| API认证 | 内部API使用Token认证 |
| CORS配置 | 限制跨域访问来源 |
| 速率限制 | API请求频率限制 |
| 输入校验 | 所有输入参数校验 |

**验收标准**:
- 未授权请求返回401
- 恶意输入被过滤
- 暴力破解被限制
