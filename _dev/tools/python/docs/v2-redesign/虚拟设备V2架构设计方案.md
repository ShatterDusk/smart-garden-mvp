### 2.2 核心组件

#### 2.2.1 Device Instance (设备实例)

```python
class DeviceInstance:
    """设备实例 - 代表一个虚拟设备"""
    
    def __init__(self, config: DeviceConfig):
        self.config = config
        self.state = DeviceState()
        self.data_generator = DataGenerator(config.scenario)
        self.network = NetworkManager(config.network)
        
    async def start(self):
        """启动设备"""
        await self.network.connect()
        await self.data_generator.start()
        
    async def stop(self):
        """停止设备"""
        await self.data_generator.stop()
        await self.network.disconnect()
```

**职责**:
- 维护设备状态（在线/离线/绑定）
- 管理数据生成器
- 管理网络连接
- 处理生命周期事件

#### 2.2.2 Device Manager (设备管理器)

```python
class DeviceManager:
    """设备管理器 - 管理多个设备实例"""
    
    def __init__(self):
        self.devices: Dict[str, DeviceInstance] = {}
        
    def create_device(self, config: DeviceConfig) -> DeviceInstance:
        """创建设备"""
        device = DeviceInstance(config)
        self.devices[device.id] = device
        return device
        
    def remove_device(self, device_id: str):
        """移除设备"""
        if device_id in self.devices:
            self.devices[device_id].stop()
            del self.devices[device_id]
```

**职责**:
- 设备生命周期管理
- 设备发现服务
- 批量操作支持

#### 2.2.3 Data Layer (数据层)

**三层数据生成策略**:

```
┌─────────────────────────────────────────┐
│         Data Generation Strategy         │
├─────────────────────────────────────────┤
│  Layer 3: Event-Driven (事件驱动)        │
│  - 浇水、断网、场景切换等事件             │
│  - 优先级最高，可覆盖其他层               │
├─────────────────────────────────────────┤
│  Layer 2: Scenario-Based (场景驱动)      │
│  - 基于当前场景的数据范围                 │
│  - 温度、湿度等基础约束                   │
├─────────────────────────────────────────┤
│  Layer 1: Natural Simulation (自然模拟)  │
│  - 物理规律模拟                          │
│  - 昼夜节律、渐变趋势、随机波动           │
└─────────────────────────────────────────┘
```

**NaturalSimulationEngine** (已实现)
- 温度：昼夜节律 + 布朗运动
- 湿度：温度负相关 + 晨露效应
- 光照：抛物线 + 云层遮挡
- 土壤：蒸发 + 植物吸水

**ScenarioEngine** (已实现)
- 8种预设场景
- 数据范围约束
- 平滑过渡

**EventTimeline** (已实现)
- 事件队列管理
- 虚拟时间系统
- 可编辑的未来事件

## 三、关键设计决策

### 3.1 异步架构

**决策**: 使用 `asyncio` 替代多线程

**理由**:
- 更好的性能（IO密集型任务）
- 更清晰的并发模型
- 更容易测试和调试

**实现**:
```python
async def main():
    device = DeviceInstance(config)
    await device.start()
    
    # 并发运行多个服务
    await asyncio.gather(
        device.run_udp_service(),
        device.run_data_generator(),
        device.run_http_client(),
    )
```

### 3.2 插件化数据生成

**决策**: 数据生成器支持插件机制

**理由**:
- 易于扩展新的模拟算法
- 支持自定义场景
- 便于单元测试

**实现**:
```python
class DataGeneratorPlugin(ABC):
    @abstractmethod
    def generate(self, context: GenerationContext) -> SensorData:
        pass

# 内置插件
class NaturalSimulationPlugin(DataGeneratorPlugin):
    """自然模拟插件"""
    
class ScenarioConstraintPlugin(DataGeneratorPlugin):
    """场景约束插件"""
    
class EventDrivenPlugin(DataGeneratorPlugin):
    """事件驱动插件"""
```

### 3.3 配置即代码

**决策**: 使用 Pydantic 进行配置验证

**理由**:
- 类型安全
- 自动验证
- 文档生成

**实现**:
```python
from pydantic import BaseModel, Field, validator

class DeviceConfig(BaseModel):
    device_id: str = Field(..., regex=r'^VIRTUAL_\d+$')
    sample_interval: int = Field(default=5, ge=1, le=60)
    
    @validator('sample_interval')
    def validate_interval(cls, v):
        if v > 60:
            raise ValueError('采样间隔不能超过60秒')
        return v
```

## 四、接口设计

### 4.1 设备控制接口

```python
class IDeviceController(ABC):
    """设备控制器接口"""
    
    @abstractmethod
    async def start(self) -> None:
        """启动设备"""
        
    @abstractmethod
    async def stop(self) -> None:
        """停止设备"""
        
    @abstractmethod
    async def bind(self, plant_id: str) -> None:
        """绑定到植物"""
        
    @abstractmethod
    async def unbind(self) -> None:
        """解绑"""
        
    @abstractmethod
    async def set_scenario(self, scenario: str) -> None:
        """设置场景"""
        
    @abstractmethod
    async def trigger_event(self, event: DeviceEvent) -> None:
        """触发事件"""
```

### 4.2 数据生成接口

```python
class IDataGenerator(ABC):
    """数据生成器接口"""

        """设置场景"""

        """触发事件"""
```

### 4.3 网络接口

```python
class INetworkManager(ABC):
```
