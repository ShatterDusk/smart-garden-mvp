## 二、目录设计原则

| 原则            | 说明                     |
| :------------ | :--------------------- |
| **数字前缀**      | 01-10 表示优先级/顺序，保持固定    |
| **中英结合**      | 目录名用英文，文档名用中文          |
| **扁平化**       | 最多3层，避免过深嵌套            |
| **README 驱动** | 每个目录必须有 README.md 作为索引 |
| **分离关注点**     | 产品、技术、任务、参考分离          |

## 四、文档管理规范

### 4.1 文档命名规范

```markdown
# 文件命名
{序号}-{描述}.md          # 文档类，如：01-需求规格说明书.md
{日期}-{功能}-spec.md     # 规格类，如：2026-04-07-环境数据-spec.md
{日期}-{功能}-tasks.md    # 任务类，如：2026-04-07-环境数据-tasks.md
{日期}-{功能}-checklist.md # 检查类，如：2026-04-07-环境数据-checklist.md

# 目录命名
01-product/               # 数字前缀+英文小写
02-architecture/
diagrams/                 # 子目录用小写
```

### 4.2 文档模板规范

每个文档必须包含头部元信息：

```markdown
# 文档标题

## 元信息
- **文档类型**: 需求/设计/流程/任务/参考
- **版本**: V1.0
- **创建日期**: 2026-04-07
- **最后更新**: 2026-04-07
- **作者**: [姓名]
- **状态**: 草稿/评审中/已发布/已归档
- **关联文档**: [链接]

---

## 正文内容...
```

### 4.3 版本管理规范

| 版本号  | 含义   | 更新时机      |
| :--- | :--- | :-------- |
| V0.x | 草稿   | 初稿编写阶段    |
| V1.0 | 首次发布 | 评审通过后     |
| V1.x | 小修订  | 内容修正、补充   |
| V2.0 | 重大更新 | 结构重构、需求变更 |

### 4.4 文档状态流转

```
草稿 → 评审中 → 已发布 → 已归档
  ↓       ↓        ↓
  └──── 废弃 ──────┘
```

### 4.5 目录 README 模板

每个目录的 README.md 必须包含：

```markdown
# 目录名

## 目录说明
简要说明本目录存放什么类型的文档。

## 文档清单
| 序号 | 文档名 | 版本 | 状态 | 最后更新 |
|:---:|:---|:---:|:---:|:---:|
| 01 | [文档名](./文档.md) | V1.0 | ✅ 已发布 | 2026-04-07 |

## 快速导航
- [返回上级](../README.md)
- [文档首页](../../README.md)
```
