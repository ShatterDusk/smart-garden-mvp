# IoT Sensor Data Simulator 使用指南

## 一、快速开始

### 1. 启动模拟器

```bash
cd f:\PROJECTS\WeChatProjects\MVP\_dev\tools\virtual_device
python app/main.py
```

访问 http://localhost:8080

---

## 二、配置流程（3步骤）

### 步骤1：创建传感器类型

1. 点击顶部 **"Sensoren"** 标签
2. 点击 **"Neuen Sensor erstellen"** (创建新传感器)
3. 填写传感器参数：

| 参数 | 说明 | 示例值 |
|:---|:---|:---|
| Name | 传感器名称 | temperature |
| Einheit | 单位 | °C |
| Basiswert | 基础值 | 25 |
| Variationsbereich | 变化范围 | 5 |
| Änderungsrate | 变化率 | 0.5 |
| Intervall (Sekunden) | 数据发送间隔(秒) | 300 |

**植物传感器配置示例**：

```
温度传感器:
  - Name: temperature
  - Einheit: °C
  - Basiswert: 25
  - Variationsbereich: 5
  - Änderungsrate: 0.5
  - Intervall: 300 (5分钟)

湿度传感器:
  - Name: humidity
  - Einheit: %
  - Basiswert: 60
  - Variationsbereich: 20
  - Änderungsrate: 2
  - Intervall: 300

土壤湿度传感器:
  - Name: soil_moisture
  - Einheit: %
  - Basiswert: 50
  - Variationsbereich: 15
  - Änderungsrate: 1
  - Intervall: 300
```

---

### 步骤2：创建设备

1. 点击顶部 **"Geräte"** 标签
2. 点击 **"Neues Gerät erstellen"** (创建新设备)
3. 输入设备名称（对应你的 DEVICE_ID）
4. 选择要关联的传感器
5. 点击 **"Erstellen"** (创建)

**设备命名建议**：
- `DEVICE_PLANT_001`
- `DEVICE_001`
- 或你的实际设备ID

---

### 步骤3：创建容器并启动模拟

1. 点击顶部 **"Container"** 标签
2. 点击 **"Neuen Container erstellen"** (创建新容器)
3. 填写容器信息：
   - Name: 容器名称（如"植物传感器容器"）
   - Beschreibung: 描述
   - Standort: 位置
4. 选择要包含的设备
5. 选择发送接口：
   - **MQTT**: 发送到 MQTT Broker
   - **IoT Hub**: 发送到 Azure IoT Hub
   - **Demo-Modus**: 仅演示，不发送数据

---

## 三、配置 MQTT（推荐）

### 1. 创建 .env 文件

在项目根目录创建 `.env` 文件：

```env
# MQTT 配置
MQTT_BROKER_ADDRESS=localhost
MQTT_BROKER_PORT=1883
MQTT_BROKER_USERNAME=your_username
MQTT_BROKER_PASSWORD=your_password
```

### 2. 容器配置

创建容器时：
- **Protokoll**: 选择 MQTT
- **Topic**: 输入你的主题（如 `plant/sensor/data`）

## 五、与你的后端集成

### 方案A：添加 MQTT 支持到你的后端（推荐）

在你的 Node.js 后端添加 MQTT 订阅：

```javascript
const mqtt = require('mqtt');
const client = mqtt.connect('mqtt://localhost:1883');

client.on('connect', () => {
  client.subscribe('plant/sensor/data');
});

client.on('message', (topic, message) => {
  const data = JSON.parse(message.toString());
  // 转换为你的 API 格式
  const apiData = {
    deviceId: data.deviceName,
    plantId: 'PLANT_001',
    timestamp: data.timestamp,
    metrics: {
      [data.sensorName]: data.value
    }
  };
  // 调用你的 service 处理数据
  deviceService.reportDeviceData(apiData);
});
```

### 方案B：修改模拟器支持 HTTP 推送

修改 `app/utils/mqtt_helper.py`，添加 HTTP 发送功能。

## 六、界面操作说明

### 顶部导航
- **Container**: 管理容器（设备组）
- **Geräte**: 管理设备
- **Sensoren**: 管理传感器类型
- **Demo-Modus**: 演示模式开关

### 容器操作按钮
- **play_arrow**: 开始模拟
- **pause**: 暂停模拟
- **insert_chart_outlined**: 查看实时数据图表
- **exit_to_app**: 导出数据

---

## 七、故障排除

### 问题1：无法连接到 MQTT
- 检查 `.env` 文件配置
- 确认 MQTT Broker 是否运行
- 检查防火墙设置

### 问题2：数据不发送
- 检查是否开启了 Demo-Modus
- 检查容器是否已启动（点击 play 按钮）
- 检查传感器间隔设置

### 问题3：页面无法访问
- 确认 `python app/main.py` 已正常运行
- 检查端口 8080 是否被占用

---

## 八、高级功能

### 错误模拟
创建传感器时可以配置错误类型：
- **Anomalie**: 异常值
- **MCAR**: 随机丢失数据
- **Duplicate Data**: 重复数据
- **Drift**: 数据漂移

### 批量导出
点击容器的 **exit_to_app** 按钮，可以导出数据为：
- CSV
- JSON
- XLSX

## 九、与你的项目对接示例

假设你的后端 API 是：
```
POST http://localhost:3000/api/devices/data
```

你需要：
1. 在设备页面创建名为 `DEVICE_001` 的设备
2. 创建温度、湿度、土壤湿度传感器
3. 将传感器关联到设备
4. 创建容器，选择该设备
5. 配置 MQTT 或修改代码支持 HTTP
6. 启动容器，开始接收数据
